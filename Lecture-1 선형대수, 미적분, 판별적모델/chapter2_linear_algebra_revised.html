도입: 왜 선형대수를 배워야 하는가? - 데이터의 기하학적 해석

머신러닝의 세계에 첫발을 내디딜 때, 우리는 종종 데이터를 엑셀 시트나 데이터베이스 테이블과 같은 숫자들의 표로 인식합니다. 그러나 이는 데이터가 가진 풍부한 의미의 일부만을 보여줄 뿐입니다. 머신러닝, 특히 선형대수의 관점에서 데이터는 단순한 숫자의 나열이 아니라, 고차원 공간에 존재하는 하나의 **점(Point)** 또는 원점으로부터 그 점을 가리키는 **벡터(Vector)**로 해석될 수 있습니다.

예를 들어, 붓꽃(Iris)의 특징인 꽃받침 길이, 꽃받침 너비, 꽃잎 길이, 꽃잎 너비라는 4개의 숫자 조합은 4차원 공간의 한 점을 정의합니다. 이처럼 데이터를 기하학적 대상으로 바라보는 순간, 우리는 새로운 관점을 얻게 됩니다. 데이터 간의 관계는 공간상 거리로, 데이터의 패턴은 점들의 분포로, 분류 문제는 이 점들을 가르는 경계면을 찾는 문제로 재정의됩니다.

선형대수는 바로 이 '데이터 공간'을 다루는 강력하고 필수적인 언어이자 도구입니다. 벡터와 행렬을 이용해 데이터를 표현하고, 행렬 연산을 통해 공간 자체를 회전시키거나 늘리고 줄이는 '변환'을 수행할 수 있습니다. 이러한 변환을 통해 우리는 데이터에 숨겨진 본질적인 구조를 파악하고, 복잡한 문제를 더 단순한 형태로 만들 수 있습니다.

본 챕터에서 다루는 벡터의 내적, Norm, 행렬의 변환, 그리고 고유값 분해와 같은 개념들은 단순히 추상적인 수학 이론에 그치지 않습니다. 이 개념들은 선형 회귀나 로지스틱 회귀와 같은 기본적인 선형 모델이 어떻게 작동하는지를 명확히 설명하는 열쇠가 됩니다. 더 나아가, 데이터의 분산을 최대로 보존하면서 차원을 축소하는 강력한 기법인 **주성분 분석(Principal Component Analysis, PCA)**의 수학적 근간을 이루게 됩니다. 이 챕터의 학습은 이후 더 복잡한 머신러닝 알고리즘을 깊이 있게 이해하기 위한 단단한 반석이 될 것입니다.

벡터와 데이터: 공간의 점으로 데이터 이해하기

이 섹션에서는 데이터의 가장 기본적인 표현 단위인 벡터에 대해 학습합니다. 벡터의 정의부터 시작하여, 벡터 간의 관계를 정량화하는 내적, 그리고 벡터의 크기와 데이터 간의 거리를 측정하는 Norm과 거리의 개념을 차례로 탐구합니다. 각 개념이 머신러닝에서 어떻게 활용되는지 연결하며, 수학적 정의와 기하학적 직관을 함께 구축하는 것을 목표로 합니다.

벡터의 정의와 표현

벡터는 크기와 방향을 동시에 가지는 물리량에서 유래한 개념으로, 수학적으로는 n개의 실수(스칼라)의 순서 있는 목록으로 정의됩니다. n차원 공간에서 벡터는 하나의 점 또는 원점에서 그 점을 가리키는 화살표로 시각화될 수 있습니다.

  • 수학적 정의: n차원 벡터 v는 다음과 같이 표현됩니다. 
    v = (v₁, v₂, ..., vₙ) ∈ ℝⁿ
    여기서 각 vᵢ는 벡터의 i번째 성분(component)이며, ℝⁿ은 n차원 실수 공간을 의미합니다.
  • 기하학적 의미: 2차원 공간에서 벡터 v = (3, 2)는 좌표평면 위의 점 (3, 2)를 나타내거나, 원점 (0, 0)에서 점 (3, 2)를 향해 뻗어 나가는 화살표로 그릴 수 있습니다. 이 화살표의 길이는 벡터의 '크기'를, 화살표가 가리키는 방향은 벡터의 '방향'을 나타냅니다.

머신러닝에서 벡터는 개별 데이터 샘플(sample)을 표현하는 핵심적인 방법입니다. 예를 들어, 붓꽃 데이터셋에서 한 개의 붓꽃 샘플은 [꽃받침 길이, 꽃받침 너비, 꽃잎 길이, 꽃잎 너비]라는 4개의 특징(feature)으로 구성됩니다. 이 샘플은 4차원 특징 공간(feature space)의 한 점, 즉 4차원 벡터로 표현될 수 있습니다. 마찬가지로, 28x28 픽셀의 흑백 이미지는 각 픽셀의 밝기 값을 하나의 성분으로 가지는 784차원(28*28=784) 벡터로 변환될 수 있습니다. 이처럼 데이터를 벡터로 변환함으로써 우리는 기하학적 분석과 대수적 연산을 데이터에 적용할 수 있게 됩니다.

벡터의 연산: 내적 (Dot Product)

내적(Dot Product 또는 Inner Product)은 두 벡터를 입력받아 스칼라 값을 출력하는 연산으로, 두 벡터 간의 관계를 파악하는 데 매우 중요한 역할을 합니다. 내적은 대수적 정의와 기하학적 정의 두 가지 방식으로 이해할 수 있습니다.

  • 수학적 정의: 두 n차원 벡터 u = (u₁, ..., uₙ)와 v = (v₁, ..., vₙ)의 내적은 각 성분을 곱한 뒤 모두 더한 값입니다. 
    u · v = u₁v₁ + u₂v₂ + ... + uₙvₙ = Σᵢ uᵢvᵢ
  • 기하학적 의미: 내적은 두 벡터의 크기와 두 벡터가 이루는 각도 θ의 코사인 값을 곱한 것과 같습니다. 
    u · v = ||u|| ||v|| cos(θ)
    여기서 ||u||와 ||v||는 각각 벡터 u와 v의 크기(Norm)를 의미합니다.

기하학적 정의에서 핵심은 cos(θ)입니다. 코사인 함수는 두 벡터의 방향이 얼마나 일치하는지를 나타내는 **유사도(Similarity)** 척도로 해석될 수 있습니다.

  • θ = 0° (cos(θ) = 1): 두 벡터가 정확히 같은 방향을 가리킬 때 내적 값은 최대가 됩니다.
  • θ = 90° (cos(θ) = 0): 두 벡터가 서로 직교(Orthogonal)할 때 내적 값은 0이 됩니다. 이는 두 벡터가 서로 독립적인 방향 성분을 가짐을 의미합니다.
  • θ = 180° (cos(θ) = -1): 두 벡터가 정확히 반대 방향을 가리킬 때 내적 값은 최소가 됩니다.

또한 내적은 **정사영(Projection)**의 개념과도 깊이 관련됩니다. 벡터 u를 벡터 v 위로 정사영시킨 벡터의 크기는 ||u|| cos(θ)이며, 이는 (u · v) / ||v||로 계산할 수 있습니다. 즉, 내적은 한 벡터가 다른 벡터 방향으로 얼마나 투영되는지를 나타냅니다.

머신러닝 및 PCA 연관성

  • 가중합(Weighted Sum): 선형 모델이나 퍼셉트론에서 예측값은 가중치 벡터 w와 입력 특성 벡터 x의 내적(w · x)으로 계산됩니다. 이는 각 특성(xᵢ)에 가중치(wᵢ)를 부여하여 중요도를 반영하는 과정으로, 내적의 대수적 정의와 일치합니다.
  • 유사도 측정: 추천 시스템에서 사용자 A의 선호도 벡터와 영화 B의 특징 벡터 간의 내적을 계산하여 사용자 A가 영화 B를 얼마나 좋아할지 예측할 수 있습니다.
  • PCA 기초 다지기: PCA는 데이터의 분산이 가장 큰 방향을 찾는 기법입니다. 데이터 포인트들을 특정 축(주성분 벡터)에 **정사영**시켰을 때, 그 정사영된 점들의 분산이 최대가 되어야 합니다. 이 정사영 계산의 핵심이 바로 내적입니다. 따라서 내적에 대한 이해는 PCA의 작동 원리를 파악하는 첫걸음입니다.

벡터의 크기와 거리: Norm & Distance

Norm(놈)은 벡터 공간에서 벡터의 '크기' 또는 '길이'를 측정하는 함수입니다. 여러 종류의 Norm이 있지만, 머신러닝에서는 주로 L2 Norm이 사용됩니다.

L2 Norm (유클리드 Norm)

  • 수학적 정의: 벡터 v = (v₁, ..., vₙ)의 L2 Norm은 각 성분의 제곱의 합에 제곱근을 취한 값입니다. 이는 벡터와 자기 자신의 내적에 제곱근을 취한 것과 같습니다: ||v||₂ = √(v · v). 
    ||v||₂ = √(v₁² + v₂² + ... + vₙ²)
  • 기하학적 의미: L2 Norm은 n차원 유클리드 공간에서 원점으로부터 벡터 v가 가리키는 점까지의 직선 거리입니다. 이는 피타고라스 정리를 n차원으로 일반화한 것입니다.
  • 머신러닝 연관성:
    • 규제(Regularization): 모델이 훈련 데이터에 과적합(overfitting)되는 것을 방지하기 위해 손실 함수에 페널티 항을 추가하는 기법입니다. L2 규제(Ridge 회귀)는 가중치 벡터 w의 L2 Norm의 제곱(||w||₂²)을 손실 함수에 더합니다. 이를 통해 가중치 값들이 너무 커지는 것을 억제하여 모델을 더 단순하고 일반화 성능이 좋게 만듭니다.
    • 오차 측정: 모델의 예측값 벡터와 실제값 벡터 간의 차이(오차 벡터)의 L2 Norm을 계산하여 모델의 전체적인 오차 크기를 측정할 수 있습니다. (예: 평균 제곱 오차(MSE)의 제곱근인 RMSE)

거리 (Distance)

두 벡터 간의 거리는 한 벡터에서 다른 벡터로의 차이 벡터의 Norm으로 정의됩니다.

  • 수학적 정의: 두 벡터 u와 v 사이의 유클리드 거리는 차이 벡터 u - v의 L2 Norm으로 계산됩니다. 
    dist(u, v) = ||u - v||₂ = √((u₁-v₁)² + (u₂-v₂)² + ... + (uₙ-vₙ)²)
  • 기하학적 의미: 특징 공간에서 두 데이터 포인트가 얼마나 떨어져 있는지를 나타내는 직관적인 척도입니다.
  • 머신러닝 연관성: k-최근접 이웃(k-NN) 알고리즘은 새로운 데이터 포인트가 주어졌을 때, 특징 공간에서 가장 가까운 k개의 훈련 데이터 포인트를 찾아 그들의 레이블을 기반으로 새로운 데이터의 레이블을 예측합니다. 여기서 '가장 가까운' 이웃을 찾는 기준이 바로 유클리드 거리입니다.

행렬과 변환: 데이터셋을 조작하고 공간을 바꾸는 도구

행렬은 단순히 숫자를 직사각형으로 배열한 것을 넘어, 데이터를 체계적으로 표현하고 공간 자체를 동적으로 변환하는 강력한 도구입니다. 이 섹션에서는 행렬을 데이터의 집합이자 공간 변환의 주체로 이해하고, 행렬의 주요 속성들이 데이터 분석, 특히 PCA와 어떻게 깊이 연결되는지 탐구합니다.

행렬의 정의와 기본 연산

행렬(Matrix)은 스칼라 값을 행(row)과 열(column)의 직사각형 구조로 배열한 것입니다. m개의 행과 n개의 열을 가진 행렬을 m x n 행렬이라고 합니다.

  • 데이터셋 표현: 머신러닝에서 데이터셋은 보통 행렬로 표현됩니다. 관례적으로 행(row)은 개별 데이터 샘플(sample)을, 열(column)은 각 샘플의 특징(feature)을 나타냅니다. 예를 들어, 150개의 붓꽃 샘플과 4개의 특징을 가진 데이터셋은 150x4 크기의 행렬 X로 표현할 수 있습니다.
  • 기본 연산: 행렬의 덧셈, 스칼라 곱, 전치(Transpose, 행과 열을 바꾸는 연산) 등은 기본적인 데이터 조작에 사용됩니다.
  • 행렬 곱셈(Matrix Multiplication): 두 행렬의 곱셈은 단순한 원소별 곱셈이 아닙니다. 행렬 A (m x n)와 B (n x p)의 곱 C = AB (m x p)는 선형 변환의 **합성(Composition of transformations)**이라는 중요한 기하학적 의미를 가집니다. 벡터 x에 행렬 B를 곱하고(Bx), 그 결과 벡터에 다시 행렬 A를 곱하는(A(Bx)) 것은, 두 변환을 순차적으로 적용하는 것과 같습니다.

핵심 개념: 행렬은 선형 변환(Linear Transformation)이다

선형대수에서 행렬의 가장 중요한 역할은 벡터를 입력받아 다른 벡터로 출력하는 **함수**, 즉 **선형 변환(Linear Transformation)**으로서의 기능입니다. 행렬은 n차원 공간의 벡터를 m차원 공간의 벡터로 매핑(mapping)하며, 이 과정에서 공간 전체를 변형시킵니다.

선형 변환은 두 가지 핵심 속성을 만족합니다:

  1. 가산성(Additivity): T(u + v) = T(u) + T(v)
  2. 동차성(Homogeneity): T(cv) = cT(v) (c는 스칼라)

이 속성들로 인해 선형 변환 후에는 원점이 그대로 원점에 머무르고, 모든 직선은 변환 후에도 여전히 직선(또는 점)으로 유지됩니다. 기하학적으로 행렬은 공간을 **회전(Rotation)**, **확대/축소(Scaling)**, **전단(Shear)**시키는 효과를 가집니다.

머신러닝 연관성

선형 모델 y = Wx + b에서 가중치 행렬 W는 입력 특징 벡터 x가 존재하는 공간을 예측값 벡터 y가 존재하는 공간으로 변환하는 선형 변환자(linear operator) 역할을 합니다. 모델을 학습한다는 것은 데이터를 가장 잘 설명하는(예측 오차를 최소화하는) 최적의 변환 행렬 W를 찾는 과정입니다. PCA 역시 데이터를 새로운 좌표계로 변환하는 과정이며, 이 변환은 특정 **직교 행렬(Orthogonal Matrix)**에 의해 수행됩니다.

행렬의 성질과 특별한 행렬들

행렬의 성질을 이해하는 것은 변환의 특징을 파악하고 데이터의 구조를 분석하는 데 필수적입니다.

  • 단위 행렬(Identity Matrix, I): 대각선 원소가 모두 1이고 나머지는 0인 정방행렬. 어떤 벡터를 변환해도 자기 자신이 나오는 '항등' 변환에 해당합니다.
  • 대각 행렬(Diagonal Matrix): 대각선 원소를 제외한 모든 원소가 0인 행렬. 각 좌표축 방향으로 독립적인 '확대/축소(scaling)' 변환을 의미합니다.
  • 역행렬(Inverse Matrix, A⁻¹): 변환을 원래대로 되돌리는(undo) 변환에 해당하는 행렬입니다 (A⁻¹A = I). 역행렬은 정방행렬이면서 행렬식이 0이 아닐 때(non-singular)만 존재합니다. 기하학적으로는 공간을 더 낮은 차원으로 찌그러뜨리지 않는(예: 2차원 평면을 선이나 점으로 만들지 않는) 변환만이 역변환을 가질 수 있습니다.
  • 랭크(Rank): 행렬 A에 의해 변환된 후의 결과 공간의 차원(dimension)입니다. 예를 들어, 3차원 공간의 점들을 어떤 행렬로 변환했더니 모두 하나의 평면 위에 놓인다면, 그 행렬의 랭크는 2입니다. 랭크가 행렬의 차원보다 낮다는 것은 데이터에 선형 종속성이 존재하며, 더 낮은 차원으로 표현될 수 있음을 시사합니다. 이는 차원 축소의 필요성과 가능성을 알려주는 중요한 단서입니다.
  • 조건수(Condition Number): 행렬이 얼마나 '불안정한지' 또는 특이 행렬(singular matrix, 역행렬이 없는 행렬)에 가까운지를 나타내는 척도입니다. 조건수가 매우 크면(ill-conditioned), 입력값의 작은 변화가 결과값에 큰 변화를 일으킬 수 있어 역행렬 계산 시 수치적 오차가 커질 수 있습니다. 머신러닝에서는 조건수가 큰 문제(예: 특징 간 다중공선성이 높은 경우)는 경사 하강법과 같은 최적화 알고리즘의 수렴을 방해하거나 매우 느리게 만들 수 있습니다.
  • 직교 행렬(Orthogonal Matrix, Q): 열벡터(또는 행벡터)들이 서로 직교하고 크기가 1인 단위 벡터들로 구성된 정방행렬입니다. QᵀQ = QQᵀ = I를 만족하며, 이는 Q⁻¹ = Qᵀ임을 의미합니다.
    • 기하학적 의미: 직교 행렬은 공간의 **회전(Rotation)** 또는 **반사(Reflection)**에 해당하는 변환입니다. 이 변환은 벡터의 **길이(Norm)와 벡터 간의 각도를 보존**합니다. 즉, 데이터의 형태나 구조를 왜곡하지 않고 오직 회전만 시킵니다.
    • PCA와의 핵심 연결: PCA는 데이터의 분산이 가장 큰 방향으로 좌표축을 '회전'시키는 과정입니다. 이때 사용되는 변환 행렬이 바로 직교 행렬입니다. 데이터의 내재적 구조를 바꾸지 않고 바라보는 관점만 바꾸는 것이죠. 또한, 역변환이 전치 행렬(Qᵀ)을 곱하는 것만으로 간단히 계산되므로, 차원 축소 후 원래 데이터로 복원하는 과정이 매우 효율적입니다.

고유값, 고유벡터, 직교기저(2D 직관)

행렬이 공간을 변환시킬 때, 대부분의 벡터는 방향이 바뀝니다. 하지만 여기서 한 가지 중요한 질문을 던질 수 있습니다: "선형 변환 후에도, 방향은 변하지 않고 오직 크기만 변하는 특별한 벡터가 존재할까?" 이 질문에 대한 답이 바로 고유벡터와 고유값입니다. 이들은 변환의 '본질'을 담고 있으며, 데이터의 숨겨진 구조를 파악하는 데 결정적인 역할을 합니다.

사례: 타원형 데이터의 주축 찾기

키와 몸무게처럼 서로 양의 상관관계를 가지는 두 변수로 이루어진 2차원 데이터가 있다고 상상해 봅시다. 이 데이터를 산점도(scatter plot)에 나타내면, 데이터 포인트들은 원형이 아닌, 한쪽으로 길게 늘어진 **타원형 구름(elliptical cloud)** 형태를 띨 가능성이 높습니다.

이 타원형 데이터 분포에서 우리는 두 가지 중요한 '축'을 생각할 수 있습니다.

  1. 주축 (Major Axis): 데이터가 가장 길게 늘어선 방향. 이 방향은 데이터의 **분산(variance)이 가장 큰 방향**입니다.
  2. 부축 (Minor Axis): 주축에 수직이면서, 데이터가 두 번째로 넓게 퍼져 있는 방향.

이 두 축을 찾는 것이 바로 **주성분 분석(PCA)**의 핵심 목표입니다. 이 축들을 찾으면, 우리는 데이터를 더 잘 이해할 수 있습니다. 예를 들어, '키와 몸무게'라는 두 가지 정보를 '체격'이라는 하나의 종합적인 정보(주축)로 요약할 수 있습니다. 즉, 2차원 데이터를 정보 손실을 최소화하면서 1차원으로 축소할 수 있게 됩니다. 이 주축을 찾는 수학적 도구가 바로 고유값과 고유벡터입니다.

이론: 대칭행렬 분해, 직교 대각화, SVD 개관

고유값과 고유벡터 (Eigenvalues and Eigenvectors)

  • 수학적 정의: 0이 아닌 벡터 v와 정방행렬 A에 대해, 다음 식을 만족하는 스칼라 λ가 존재할 때, v를 A의 **고유벡터(Eigenvector)**라 하고, λ를 그에 대응하는 **고유값(Eigenvalue)**이라고 합니다. 
    Av = λv
  • 기하학적 의미:
    • 고유벡터(v): 행렬 A가 나타내는 선형 변환의 '축(axis)'입니다. 이 축 방향의 벡터들은 변환 후에도 원래 방향을 그대로 유지합니다. (방향이 반대가 되는 경우도 포함)
    • 고유값(λ): 해당 고유벡터 방향으로 공간이 얼마나 늘어나거나(λ > 1), 줄어드는지(0 < λ < 1), 또는 방향이 뒤집히는지(λ < 0)를 나타내는 **스케일링 비율(scaling factor)**입니다.

대칭행렬 분해와 스펙트럼 정리 (Symmetric Matrix Decomposition & Spectral Theorem)

데이터의 공분산 행렬은 항상 **대칭 행렬(Symmetric Matrix, A = Aᵀ)**입니다. 대칭 행렬은 다음과 같은 매우 중요한 성질을 가집니다.

  • 모든 고유값(λ)은 실수(real number)입니다.
  • 서로 다른 고유값에 대응하는 고유벡터들은 항상 서로 **직교(orthogonal)**합니다.

이러한 성질 덕분에, 대칭 행렬 A는 항상 **고유값 분해(Eigendecomposition)**가 가능하며, 이는 **스펙트럼 정리(Spectral Theorem)**로 알려져 있습니다. 스펙트럼 정리는 대칭 행렬 A가 다음과 같이 세 행렬의 곱으로 분해될 수 있음을 보장합니다.

A = PDP⁻¹ = PDPᵀ
  • P: A의 고유벡터들을 열벡터로 가지는 **직교 행렬(Orthogonal Matrix)**입니다. (P⁻¹ = Pᵀ)
  • D: A의 고유값들을 대각 원소로 가지는 **대각 행렬(Diagonal Matrix)**입니다.

이 과정을 **직교 대각화(Orthogonal Diagonalization)**라고 합니다. 이는 복잡한 선형 변환 A를 '좌표축 회전(Pᵀ)', '각 축 방향으로의 독립적인 스케일링(D)', '원래대로 회전(P)'이라는 세 단계의 간단한 기하학적 변환으로 나누어 해석할 수 있게 해줍니다. PCA에서 공분산 행렬을 분해하는 것이 바로 이 직교 대각화 과정입니다.

특이값 분해 (Singular Value Decomposition, SVD) 개관

고유값 분해는 정방행렬에만 적용할 수 있다는 한계가 있습니다. **특이값 분해(SVD)**는 이를 일반화하여, 정방행렬이 아닌 임의의 m x n 행렬 A에 대해서도 적용할 수 있는 강력한 행렬 분해 기법입니다.

A = UΣVᵀ
  • U: m x m 크기의 직교 행렬 (좌측 특이벡터)
  • Σ: m x n 크기의 대각 행렬 (대각 원소는 특이값(singular value)이며, 0 이상)
  • V: n x n 크기의 직교 행렬 (우측 특이벡터)

SVD는 PCA와 매우 밀접한 관련이 있습니다. 데이터 행렬 X의 공분산 행렬 XᵀX의 고유벡터는 X의 SVD에서 V 행렬의 열벡터들과 같고, 고유값은 특이값의 제곱과 비례합니다. 따라서 SVD를 통해서도 PCA의 주성분을 찾을 수 있으며, 수치적으로 더 안정적이기 때문에 많은 PCA 구현에서 내부적으로 SVD를 사용합니다.

예제: 2x2 공분산의 주성분 방향 선택

어떤 2차원 데이터의 공분산 행렬 C가 다음과 같이 주어졌다고 가정합시다. 이 행렬의 고유값과 고유벡터를 계산하여 데이터의 주성분 방향을 찾아봅시다.

C = [ 5  2 ]
    [ 2  2 ]
  1. 고유값(Eigenvalue) 계산: 특성 방정식 det(C - λI) = 0을 풉니다. 
    det( [ 5-λ  2 ] ) = (5-λ)(2-λ) - (2)(2) = 0
    [ 2   2-λ]

λ² - 7λ + 10 - 4 = 0
λ² - 7λ + 6 = 0
(λ - 6)(λ - 1) = 0
    따라서 고유값은 λ₁ = 6, λ₂ = 1 입니다.
  2. 고유벡터(Eigenvector) 계산: 각 고유값에 대해 (C - λI)v = 0을 풉니다.
    • λ₁ = 6 일 때: 
      (C - 6I)v₁ = [ -1  2 ] [x] = [0]
                   [  2 -4 ] [y]   [0]
      -x + 2y = 0 이므로, x = 2y 입니다. 따라서 고유벡터 v₁은 [2, 1] 방향의 벡터입니다. (예: v₁ = [2, 1]ᵀ)
    • λ₂ = 1 일 때: 
      (C - 1I)v₂ = [ 4  2 ] [x] = [0]
                   [ 2  1 ] [y]   [0]
      2x + y = 0 이므로, y = -2x 입니다. 따라서 고유벡터 v₂는 [1, -2] 방향의 벡터입니다. (예: v₂ = [1, -2]ᵀ)
  3. 결과 해석 및 PCA와의 연결:
    • 주성분 방향: 가장 큰 고유값 λ₁ = 6에 해당하는 고유벡터 v₁ = [2, 1]ᵀ이 바로 **제1 주성분(PC1)**의 방향입니다. 이는 데이터의 분산이 가장 큰 방향을 의미합니다. 두 번째로 큰 고유값 λ₂ = 1에 해당하는 고유벡터 v₂ = [1, -2]ᵀ는 **제2 주성분(PC2)**의 방향이며, PC1에 직교합니다.
    • 분산의 크기: 고유값의 크기는 해당 주성분 방향으로의 데이터 분산의 크기를 나타냅니다. 이 데이터의 총 분산은 λ₁ + λ₂ = 6 + 1 = 7이며, 이 중 약 85.7% (6/7)의 분산이 PC1 방향에 의해 설명됩니다.
    • 차원 축소: 만약 이 2차원 데이터를 1차원으로 축소해야 한다면, 정보 손실을 최소화하기 위해 PC1, 즉 v₁ = [2, 1]ᵀ 방향의 축에 데이터를 정사영시키는 것이 최적의 선택입니다.

이처럼 공분산 행렬의 고유값 분해는 데이터의 숨겨진 주축과 그 중요도를 명확하게 알려주며, 이는 PCA를 통한 효과적인 차원 축소의 수학적 기반이 됩니다.
