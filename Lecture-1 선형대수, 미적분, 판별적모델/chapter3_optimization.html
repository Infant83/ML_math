챕터 3: 미분과 최적화 기초

도입: 왜 머신러닝에서 최적화가 중요한가?

머신러닝 모델의 '학습' 과정은 본질적으로 '최적화(Optimization)' 문제를 푸는 것과 같습니다. 모델이 주어진 데이터에 가장 잘 부합하도록 만드는 것은, 모델의 예측과 실제 값 사이의 오차를 나타내는 손실 함수(Loss Function)의 값을 최소화하는 최적의 파라미터(가중치) 집합을 찾는 여정입니다. 이 과정은 마치 안개 낀 산속에서 가장 낮은 계곡 지점을 찾아 내려가는 것과 비유할 수 있습니다.

이때, 우리가 어느 방향으로 발걸음을 옮겨야 가장 효율적으로 하산할 수 있을지 알려주는 나침반 역할을 하는 것이 바로 미분(Differentiation)입니다. 미분은 특정 지점에서 함수의 순간적인 변화율, 즉 기울기를 알려주어 우리가 가야 할 가장 가파른 내리막길의 방향을 제시합니다. 손실 함수라는 복잡한 지형도 위에서, 미분은 우리가 나아갈 방향과 보폭을 결정하는 핵심적인 수학적 도구입니다.

본 챕터에서는 손실 함수의 지형을 이해하기 위한 다변수 미분 개념, 즉 그래디언트(Gradient)와 헤시안(Hessian)을 학습합니다. 이를 바탕으로, 가장 효율적인 하산길을 찾는 다양한 최적화 알고리즘, 대표적으로 경사하강법(Gradient Descent)과 뉴턴법(Newton's Method)의 원리를 수학적 이론과 직관적인 시각화를 통해 깊이 있게 탐구하는 것을 목표로 합니다.

다변수 미분: 손실 함수의 지형도 읽기

손실 함수는 일반적으로 수많은 파라미터를 변수로 갖는 다변수 함수입니다. 이러한 고차원 함수의 형태를 분석하고 최저점을 찾기 위해서는 일변수 함수를 다룰 때보다 더 정교한 도구가 필요합니다. 여기서는 손실 함수의 '지형도'를 읽기 위한 핵심 도구인 그래디언트와 헤시안에 대해 학습합니다.

사례: 2차원 산의 가장 낮은 계곡을 찾아서

가장 간단한 형태의 다변수 함수인 2변수 이차함수를 통해 최적화의 기본 개념을 시각적으로 이해해 보겠습니다. 예를 들어, f(x, y) = x² + y²와 같은 함수를 생각해 봅시다. 이 함수의 3D 그래프는 마치 그릇 모양과 같으며, 등고선(Contour Plot)은 원점을 중심으로 하는 동심원들로 나타납니다. 최저점은 명백히 원점 (0, 0)입니다.

여기서 두 가지 중요한 직관을 얻을 수 있습니다.

  • 등고선과 그래디언트: 특정 지점에서 등고선에 수직인 방향은 경사가 가장 가파른 방향입니다. 이 방향이 바로 그래디언트 벡터(∇f)가 가리키는 방향입니다. 따라서 최저점을 향해 가장 빠르게 내려가는 길은 그래디언트의 반대 방향, 즉 음의 그래디언트(-∇f) 방향임을 알 수 있습니다.
  • 계곡의 모양과 헤시안: 만약 함수가 f(x, y) = x² + 10y²처럼 변수 앞의 계수가 다르다면, 등고선은 한쪽으로 길쭉한 타원 형태가 됩니다. 이는 각 방향으로의 '곡률(Curvature)', 즉 휘어진 정도가 다름을 의미합니다. 이처럼 함수 표면의 곡률 정보를 담고 있는 것이 바로 헤시안 행렬(Hessian Matrix)입니다. 곡률은 최적화 알고리즘의 수렴 속도에 지대한 영향을 미칩니다.

이론: 그래디언트와 헤시안의 수학적 의미

다변수 미분과 그래디언트 (Gradient, ∇f)

그래디언트는 다변수 함수를 각 변수로 편미분한 결과를 벡터 형태로 나타낸 것입니다. n개의 변수 x = (x₁, x₂, ..., xₙ)를 갖는 함수 f(x)의 그래디언트는 다음과 같이 정의됩니다.

∇f(x) = [ ∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ ]ᵀ

기하학적으로 그래디언트 벡터 ∇f(a)는 특정 지점 a에서 함수 값을 가장 빠르게 증가시키는 방향을 가리키며, 그 벡터의 크기(norm) ||∇f(a)||는 해당 방향으로의 변화율을 나타냅니다. 따라서 최적화에서는 함수 값을 감소시켜야 하므로, 그래디언트의 반대 방향 -∇f(a)로 파라미터를 업데이트합니다.

테일러 근사 (Taylor Approximation)

복잡한 함수를 특정 지점 근처에서 더 간단한 다항함수로 근사하는 것은 최적화 알고리즘을 이해하는 데 매우 중요합니다.

  • 1차 테일러 근사: 점 a 근처에서 함수 f(x)는 선형 함수로 근사할 수 있습니다. 이는 경사하강법의 이론적 기반이 됩니다.
    f(x) ≈ f(a) + ∇f(a)ᵀ(x-a)
  • 2차 테일러 근사: 2차 정보까지 사용하면 함수를 이차 함수로 더 정밀하게 근사할 수 있습니다. 이는 뉴턴법의 이론적 기반이 됩니다.
    f(x) ≈ f(a) + ∇f(a)ᵀ(x-a) + ½(x-a)ᵀH(a)(x-a)
헤시안 행렬 (Hessian Matrix, H)

헤시안 행렬은 함수의 모든 이계 편미분(second-order partial derivatives)을 행렬 형태로 나타낸 것입니다. 이는 함수 표면의 '곡률(curvature)'에 대한 정보를 담고 있습니다.

Hij = ∂²f / ∂xᵢ∂xⱼ

헤시안은 그래디언트가 각 방향으로 얼마나 변하는지를 측정하여, 함수가 해당 지점에서 위로 볼록한지(최소점 후보), 아래로 볼록한지(최대점 후보), 아니면 방향에 따라 볼록성이 변하는 안장점(saddle point) 형태인지를 결정합니다.

헤시안과 임계점(Critical Point) 판정

그래디언트가 0인 지점∇f=0), 즉 임계점에서 헤시안 행렬의 고유값(eigenvalues)을 통해 그 지점의 성격을 판별할 수 있습니다.

  • 양의 정부호 (Positive Definite): 헤시안의 모든 고유값이 양수이면, 그 지점은 지역 최소점(local minimum)입니다. 모든 방향에서 위로 볼록한 그릇 모양입니다.
  • 음의 정부호 (Negative Definite): 모든 고유값이 음수이면, 지역 최대점(local maximum)입니다.
  • 부정부호 (Indefinite): 양수와 음수 고유값을 모두 가지면, 안장점(saddle point)입니다. 말안장처럼 어떤 방향에서는 최소값, 다른 방향에서는 최대값을 가집니다.

예제: 그래디언트와 헤시안 직접 계산하기

함수 f(x, y) = 2x² + 3xy + y²가 주어졌을 때, 그래디언트와 헤시안을 계산해 봅시다.

  1. 그래디언트 계산:
    • ∂f/∂x = 4x + 3y
    • ∂f/∂y = 3x + 2y

    따라서 그래디언트 벡터는 다음과 같습니다.

    ∇f(x, y) = [ 4x + 3y, 3x + 2y ]ᵀ
  2. 헤시안 행렬 계산:
    • ∂²f/∂x² = 4
    • ∂²f/∂y² = 2
    • ∂²f/∂x∂y = 3
    • ∂²f/∂y∂x = 3

    따라서 헤시안 행렬은 다음과 같습니다.

    H = [ 4 3 ]
    [ 3 2 ]
  3. 특정 점 (1, 1)에서의 해석:
    • ∇f(1, 1) = [7, 5]ᵀ: 점 (1, 1)에서는 (7, 5) 방향으로 함수 값이 가장 가파르게 증가합니다.
    • 헤시안 행렬 H의 고유값을 계산하면 (det(H-λI) = (4-λ)(2-λ)-9 = λ²-6λ-1=0), λ는 3 ± √10 입니다. 양수와 음수 고유값을 모두 가지므로, 이 함수는 원점(임계점)에서 안장점을 가집니다.

수치 최적화 1: 경사하강법 (Gradient Descent)

경사하강법은 손실 함수의 최저점을 찾기 위해 가장 널리 사용되는 기본 최적화 알고리즘입니다. 그래디언트가 가리키는 방향의 반대쪽으로 조금씩 이동하는 과정을 반복하여 점진적으로 최소값에 근접합니다.

사례: 길고 좁은 계곡에서 지그재그로 헤매는 등산가

손실 함수의 등고선이 한쪽으로 길쭉한 타원형(예: f(x,y) = x² + 25y²)일 경우를 상상해 봅시다. 이런 지형은 '조건수(Condition Number)'가 높다고 표현하며, 최적화하기 까다로운 문제입니다. 일반적인 경사하강법(GD)은 매 스텝마다 현재 위치에서 가장 가파른 방향(등고선에 수직인 방향)으로만 이동하기 때문에, 좁고 긴 계곡의 양쪽 벽을 반복적으로 부딪히며 지그재그 형태로 매우 느리게 하산하게 됩니다.

반면, 모멘텀(Momentum)을 적용한 경사하강법은 이전에 이동하던 방향과 속도를 '관성'처럼 유지하려는 성질이 있습니다. 이 관성 덕분에 좌우로 진동하는 움직임은 상쇄되고, 계곡의 바닥을 따라 내려가는 주된 흐름은 가속되어 훨씬 더 빠르고 부드럽게 최저점에 도달할 수 있습니다.

이론: 경사하강법의 작동 원리와 한계

경사하강법 알고리즘

경사하강법의 업데이트 규칙은 매우 간단합니다. 현재 파라미터 위치 w_t에서 손실 함수 f(w)의 그래디언트 ∇f(w_t)를 계산하고, 그 반대 방향으로 학습률(learning rate) η만큼 이동하여 다음 위치 w_{t+1}을 결정합니다.

wt+1 = wt - η∇f(wt)
학습률 (Learning Rate, η)의 중요성

학습률 η는 한 번에 얼마나 이동할지, 즉 보폭을 결정하는 매우 중요한 하이퍼파라미터입니다.

  • 너무 큰 학습률: 최저점을 지나쳐 반대편으로 건너뛰는 현상이 반복되면서 값이 점점 커져 발산(diverge)할 수 있습니다.
  • 너무 작은 학습률: 수렴은 보장되지만, 최저점까지 도달하는 데 너무 많은 시간이 걸려 학습이 비효율적입니다.

최적의 학습률을 찾는 것은 '라인 서치(Line Search)'와 같은 이론적 방법론이 있지만, 실제로는 경험적으로 여러 값을 시도해보며 결정하는 경우가 많습니다.

수렴률과 조건수 (Condition Number, κ)

최적화 문제의 '난이도'를 나타내는 척도가 바로 조건수(κ)입니다. 이는 손실 함수의 헤시안 행렬의 가장 큰 고유값λ_max)과 가장 작은 고유값λ_min)의 비율로 정의됩니다.

κ = λmax / λmin

조건수는 등고선의 형태와 직접적인 관련이 있습니다. κ=1이면 등고선이 원형에 가까워 어느 방향에서 시작하든 최저점을 향해 직선으로 빠르게 수렴합니다. 반면, κ가 매우 크면 등고선이 길쭉한 타원형이 되어, 대부분의 방향에서 그래디언트가 실제 최저점이 아닌, 계곡의 가파른 벽 쪽을 향하게 됩니다. 이로 인해 지그재그 현상이 발생하며 수렴이 매우 느려집니다.

모멘텀 (Momentum)

모멘텀은 이러한 지그재그 현상을 완화하기 위해 제안된 기법입니다. 핵심 아이디어는 '관성'을 도입하는 것입니다. 즉, 과거에 이동했던 방향을 일정 비율(β)만큼 기억하고, 이를 현재 스텝의 이동 방향에 더해줍니다.

vt = βvt-1 + ∇f(wt)
wt+1 = wt - ηvt

여기서 v_t는 이동 방향과 속도를 담은 '속도' 벡터입니다. 이 방식은 그래디언트의 방향이 자주 바뀌는 축(좁은 계곡의 벽)에서는 이전의 반대 방향 이동과 현재 이동이 상쇄되어 진동을 줄여주고, 그래디언트 방향이 일관된 축(계곡의 바닥)에서는 이동이 누적되어 가속되는 효과를 낳습니다.

예제: 학습률에 따른 수렴/발산 판단

1차원 이차함수 f(x) = 5x²에 대해 시작점 x₀ = 10에서 경사하강법을 적용해 봅시다.

  1. 그래디언트 계산: ∇f(x) = f'(x) = 10x
  2. 업데이트 규칙 유도: x_{t+1} = x_t - η(10x_t) = (1 - 10η)x_t
  3. 수렴 조건: 수렴하기 위해서는 공비 |1 - 10η|가 1보다 작아야 합니다.

    -1 < 1 - 10η < 1

    -2 < -10η < 0

    0 < 10η < 2

    따라서 수렴을 위한 학습률의 범위는 0 < η < 0.2 입니다.

  4. 학습률에 따른 계산:
    • η = 0.1 (수렴): x_{t+1} = (1 - 1)x_t = 0. 한 번에 최저점으로 수렴합니다.

      x₁ = 0

    • η = 0.25 (발산): x_{t+1} = (1 - 2.5)x_t = -1.5x_t.

      x₁ = -1.5 * 10 = -15

      x₂ = -1.5 * (-15) = 22.5

      x₃ = -1.5 * 22.5 = -33.75

      값이 점점 진동하며 커지는 것을 볼 수 있습니다.

수치 최적화 2: 뉴턴법 (Newton's Method)

뉴턴법은 경사하강법보다 훨씬 정교한 최적화 기법입니다. 1차 정보인 기울기(그래디언트)뿐만 아니라 2차 정보인 곡률(헤시안)까지 활용하여 손실 함수의 최저점을 훨씬 빠르고 정확하게 찾아냅니다.

사례: 한 걸음에 계곡 중심으로 점프하기

경사하강법이 현재 위치에서 가장 가파른 내리막길로 한 걸음씩 나아가는 '근시안적인' 방법이라면, 뉴턴법은 주변 지형 전체를 조망하고 가장 낮은 지점으로 '점프'하는 것과 같습니다. 동일한 타원형 등고선 위에서 시작하더라도, 경사하강법은 등고선에 수직인 방향으로 조금 이동하는 반면, 뉴턴법은 현재 지점 주변의 함수를 이차함수로 근사한 뒤, 그 근사된 이차함수의 최저점으로 한 번에 이동합니다. 대부분의 경우, 이 한 번의 점프가 경사하강법의 수십, 수백 걸음보다 훨씬 더 목표점에 가깝게 도달합니다.

이론: 2차 근사를 활용한 최적화

뉴턴법의 핵심 아이디어

뉴턴법의 핵심은 손실 함수 f(w)를 현재 지점 w_t 근처에서 2차 테일러 근사를 통해 이차함수로 모델링하는 것입니다. 그 후, 실제 함수 f(w)를 최소화하는 대신, 이 간단한 근사 이차함수를 최소화하는 지점을 다음 스텝 w_{t+1}로 삼습니다.

업데이트 규칙

2차 테일러 근사식 f(w) ≈ f(w_t) + ∇f(w_t)ᵀ(w-w_t) + ½(w-w_t)ᵀH(w_t)(w-w_t)을 w에 대해 미분하여 0으로 놓으면 근사 함수의 최저점을 찾을 수 있습니다.

∇f(w_t) + H(w_t)(w-w_t) = 0

이 식을 w에 대해 정리하면 뉴턴법의 업데이트 규칙이 유도됩니다.

wt+1 = wt - H(wt)⁻¹∇f(wt)

여기서 H(w_t)⁻¹∇f(w_t)를 '뉴턴 스텝(Newton step)'이라고 부릅니다. 이는 그래디언트를 헤시안의 역행렬로 변환하여 이동 방향과 거리를 조절하는 것을 의미합니다. 이 변환은 길고 좁은 계곡 지형을 원형 지형으로 바꾸어주는 효과가 있어 지그재그 문제를 근본적으로 해결합니다.

장점과 단점 (계산 비용 트레이드오프)

뉴턴법은 강력하지만 현실적인 제약이 따릅니다.

  • 장점:
    • 빠른 수렴 속도: 최저점 근처에서 수렴 속도가 매우 빠릅니다(이차 수렴, Quadratic convergence).
    • 학습률 불필요: 이상적으로는 학습률 같은 하이퍼파라미터 튜닝이 필요 없습니다.
  • 단점:
    • 높은 계산 비용: 파라미터 개수가 n일 때, 헤시안 행렬을 계산하는 데 O(n²), 그 역행렬을 계산하는 데 O(n³)의 엄청난 계산 비용이 발생합니다. 파라미터가 수백만 개 이상인 현대 딥러닝 모델에는 사실상 적용이 불가능합니다.
    • 안정성 문제: 헤시안이 양의 정부호가 아니거나 (안장점 근처), 특이 행렬(singular)에 가까우면 뉴턴 스텝이 불안정해지거나 발산할 수 있습니다.
준뉴턴법 (Quasi-Newton Method) 직관

뉴턴법의 엄청난 계산량을 해결하기 위한 대안으로 준뉴턴법이 제안되었습니다. 그중 가장 유명한 것이 BFGS(Broyden–Fletcher–Goldfarb–Shanno) 알고리즘입니다.

BFGS의 핵심 아이디어는 실제 헤시안 역행렬(H⁻¹)을 매 스텝마다 직접 계산하는 대신, 이전 스텝의 파라미터 변화량과 그래디언트 변화량 정보를 이용하여 H⁻¹을 점진적으로 '근사(approximate)'하는 것입니다. 이는 헤시안을 직접 계산하지 않으면서도 뉴턴법에 가까운 빠른 수렴 속도를 얻을 수 있게 해주는 매우 효율적인 방법입니다.

예제: 뉴턴 업데이트 1회 계산해보기

함수 f(x, y) = x² + 4y²에 대해 시작점 w₀ = (2, 1)ᵀ에서 뉴턴 업데이트를 1회 수행해 봅시다.

  1. 그래디언트 계산:

    ∇f(x, y) = [2x, 8y]ᵀ

    ∇f(w₀) = ∇f(2, 1) = [4, 8]ᵀ

  2. 헤시안 행렬 계산:
    H = [ 2 0 ]
    [ 0 8 ]

    (이 예제에서는 헤시안이 상수 행렬입니다.)

  3. 헤시안 역행렬 계산:
    H⁻¹ = [ 1/2 0 ]
    [ 0 1/8 ]
  4. 뉴턴 업데이트:

    w₁ = w₀ - H⁻¹∇f(w₀)

    w₁ = [2, 1]ᵀ - [1/2, 0; 0, 1/8] * [4, 8]ᵀ

    w₁ = [2, 1]ᵀ - [2, 1]ᵀ = [0, 0]ᵀ

이차함수의 경우, 뉴턴법은 단 한 번의 스텝으로 정확히 최저점 (0, 0)에 도달합니다. 이는 뉴턴법이 함수의 곡률 정보를 얼마나 효과적으로 사용하는지를 보여주는 강력한 예시입니다.

요약 및 다음 챕터 예고

핵심 정리

  • 그래디언트(∇f): 손실 함수라는 지형에서 '어느 방향'으로 가야 가장 가파른지를 알려주는 나침반입니다.
  • 헤시안(H): '지형이 얼마나, 어떻게 휘었는지(곡률)'를 알려주는 정교한 지형도입니다.
  • 경사하강법(GD): 나침반(그래디언트)만 보고 현재 위치에서 가장 가파른 내리막길로 한 걸음씩 가는 가장 기본적인 최적화 방법입니다. 모멘텀은 여기에 '관성'을 더해 효율을 높인 개선판입니다.
  • 뉴턴법: 나침반(그래디언트)과 지형도(헤시안)를 모두 활용하여 최적의 경로로 '점프'하는 매우 빠르지만 계산 비용이 비싼 방법입니다.

지금까지 우리는 머신러닝 모델을 학습시키는 과정의 핵심인 '최적화'의 수학적 원리를 탐구했습니다. 손실 함수의 최저점을 찾아가는 이 여정은 미분이라는 도구를 통해 가능해집니다. 그래디언트는 우리에게 방향을 제시하고, 헤시안은 그 길의 험난함을 알려줍니다. 경사하강법과 뉴턴법은 이 정보들을 활용하여 최적의 파라미터를 찾는 각기 다른 전략입니다.

다음 챕터에서는 이러한 최적화 원리가 구체적인 판별 모델에서 어떻게 작동하는지 살펴볼 것입니다. 로지스틱 회귀(Logistic Regression)와 서포트 벡터 머신(Support Vector Machine)과 같은 모델들이 각각의 손실 함수를 정의하고, 오늘 배운 최적화 알고리즘을 통해 어떻게 데이터 클래스를 구분하는 '결정 경계(Decision Boundary)'를 학습해 나가는지 직접 확인하게 될 것입니다.
