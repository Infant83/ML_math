챕터 2: 선형대수 기초와 데이터의 기하학적 이해

도입: 왜 선형대수가 머신러닝의 언어인가?

머신러닝을 처음 접할 때 우리는 복잡한 알고리즘과 방대한 데이터의 세계에 압도되곤 합니다. 하지만 이 모든 것의 근간에는 데이터를 표현하고, 그 관계를 해석하며, 숨겨진 패턴을 찾아내는 일관된 '언어'가 존재합니다. 그 언어가 바로 **선형대수(Linear Algebra)**입니다.

선형대수는 단순히 행렬과 벡터를 이용한 계산 기술이 아닙니다. 이는 데이터를 기하학적 대상으로 바라볼 수 있게 해주는 강력한 관점입니다. 예를 들어, 우리가 다루는 고양이 사진, 고객 리뷰 텍스트, 주택 가격 데이터 등은 모두 고차원 공간에 존재하는 하나의 '점(point)' 또는 '벡터(vector)'로 표현될 수 있습니다. 머신러닝 모델의 학습 과정은 이 데이터 포인트들이 놓인 공간의 구조를 이해하고, 그들을 가장 잘 구분하거나 예측하는 최적의 '경계면(boundary)'을 찾는 여정과 같습니다.

이 챕터에서는 다음과 같은 질문에 대한 답을 찾아갑니다:

  • 데이터는 어떻게 공간 속 벡터로 변환되는가?
  • 모델이 학습하는 '결정 경계'는 기하학적으로 어떤 의미를 가지는가?
  • 데이터를 회전시키고, 늘리고, 압축하는 변환은 어떻게 이루어지며, 이는 어떤 통찰을 주는가?
  • 데이터에 내재된 가장 중요한 '방향' 또는 '축'은 어떻게 찾아낼 수 있는가?

이번 챕터의 학습을 통해, 우리는 추상적인 수식과 알고리즘을 넘어 데이터가 살아 숨 쉬는 '공간'을 직접 탐험하게 될 것입니다. 선형대수라는 렌즈를 통해 데이터의 본질을 꿰뚫어 보고, 머신러닝 모델의 작동 원리를 더욱 깊이 있게 이해하는 것을 목표로 합니다.

벡터와 공간: 머신러닝의 기본 단위

머신러닝의 모든 것은 데이터를 수학적으로 표현하는 것에서 시작합니다. 벡터는 이 표현의 가장 기본이 되는 단위이며, 벡터들이 모여 이루는 공간은 머신러닝 알고리즘이 작동하는 무대가 됩니다. 이 절에서는 벡터의 기본 개념부터 시작하여, 이를 통해 머신러닝의 핵심 요소인 입력 공간, 가설, 손실 함수를 기하학적으로 이해해 보겠습니다.

1. 이론 1: 데이터의 벡터 표현과 입력 공간(Input Space)

벡터(Vector)는 크기와 방향을 가진 물리량으로 정의되지만, 데이터 과학에서는 주로 숫자의 정렬된 목록, 즉 '특징(feature)'의 집합으로 사용됩니다. 예를 들어, 어떤 사람의 건강 정보를 (키, 몸무게, 나이)로 나타낸다면, 이는 [175, 70, 30]과 같은 3차원 벡터가 됩니다. 대수적으로는 숫자의 순서쌍 x = [x₁, x₂, ..., xₙ]으로, 기하학적으로는 원점에서 해당 좌표를 향하는 화살표로 시각화할 수 있습니다.

입력 공간(Input Space)은 가능한 모든 데이터 벡터 x가 존재하는 n차원 공간(일반적으로 ℝⁿ)을 의미합니다. 특징이 2개(예: 귀 길이, 코 길이)라면 2차원 평면이 입력 공간이 되고, 특징이 100개라면 100차원 공간이 됩니다. 머신러닝 모델은 이 입력 공간에 분포된 데이터 포인트들로부터 패턴을 학습합니다.

라벨(Label)은 각 데이터 벡터 x에 할당된 정답 값 y를 말합니다. 분류 문제에서는 '개', '고양이'와 같은 범주가 되며, 이를 보통 숫자로 변환하여 사용합니다(예: 개는 +1, 고양이는 -1). 회귀 문제에서는 주택 가격과 같은 연속적인 값이 라벨이 됩니다.

2. 이론 2: 벡터 연산의 기하학적 의미 - 내적, Norm, 거리

벡터 간의 연산은 데이터 포인트들 사이의 관계를 정량화하는 데 필수적입니다.

  • 내적(Dot Product): 두 벡터 w와 x의 내적은 w · x = wᵀx = Σ wᵢxᵢ로 계산됩니다. 기하학적으로 내적은 ||w|| ||x|| cos(θ)와 같습니다. 여기서 θ는 두 벡터 사이의 각도입니다. 이는 두 벡터가 얼마나 같은 방향을 향하는지, 즉 '유사도'를 측정하는 척도로 해석될 수 있습니다. cos(θ)가 1에 가까울수록(각도가 0에 가까울수록) 두 벡터는 유사합니다. 머신러닝에서 가중치 벡터 w와 입력 벡터 x의 내적은 입력이 특정 패턴w가 나타내는)과 얼마나 일치하는지를 계산하는 핵심 연산입니다.
  • Norm: 벡터의 '크기' 또는 '길이'를 나타냅니다. 가장 널리 쓰이는 L2-Norm은 유클리드 거리(Euclidean distance)로, 원점으로부터 벡터 x까지의 거리를 의미합니다. 수식은 ||x||₂ = √(Σ xᵢ²) 입니다.
  • 거리(Distance): 입력 공간 내 두 데이터 포인트 xᵢ와 xⱼ 사이의 거리는 dist(xᵢ, xⱼ) = ||xᵢ - xⱼ||로 계산됩니다. 이는 두 데이터가 얼마나 유사한지를 측정하는 가장 직관적인 방법이며, K-최근접 이웃(K-NN)과 같은 알고리즘의 기반이 됩니다.

3. 사례 연구: 고양이/개 분류를 위한 결정 경계(Decision Boundary) 스케치

머신러닝의 분류 문제를 기하학적으로 이해하기 위해 간단한 시나리오를 가정해 봅시다. '귀 길이'와 '코 길이'라는 두 가지 특징으로 고양이와 개를 구분한다고 상상해 보세요. 이 데이터는 2차원 입력 공간에 점으로 표현될 수 있습니다.

위 그림처럼, 우리는 녹색 원(개)과 빨간 사각형(고양이) 데이터 포인트들을 2차원 평면에 나타낼 수 있습니다. 두 클래스를 구분하기 위해 우리는 자연스럽게 그들 사이에 선을 긋게 됩니다. 이 '선'이 바로 **결정 경계(Decision Boundary)**입니다. 머신러닝 분류 모델이 학습한다는 것은, 주어진 데이터를 가장 잘 나눌 수 있는 최적의 결정 경계를 수학적으로 찾아내는 과정입니다.

4. 이론 3: 모델의 학습 과정을 기하학으로 이해하기

결정 경계를 찾는 과정을 더 깊이 이해하기 위해 몇 가지 개념을 정의해야 합니다.

  • 가설(Hypothesis): 특정 결정 경계를 정의하는 함수를 의미합니다. 선형 분류기에서 가설은 h(x) = w · x + b 형태의 함수입니다. 이 함수의 값이 0이 되는 지점들의 집합이 바로 결정 경계가 됩니다.
  • 가설 공간(Hypothesis Space): 모델이 탐색할 수 있는 모든 가능한 가설들의 집합입니다. 선형 분류기의 경우, 이는 모든 가능한 가중치 w와 편향 b에 의해 정의되는 모든 가능한 선(또는 초평면)들의 집합을 의미합니다. 학습 알고리즘은 이 광대한 가설 공간에서 최적의 가설 하나를 선택하는 과정입니다.
  • 손실 함수(Loss Function): 특정 가설이 주어진 데이터를 얼마나 '잘못' 분류하는지를 정량적으로 측정하는 함수입니다. 예를 들어, 0-1 손실은 예측이 틀리면 1, 맞으면 0의 손실을 부여합니다. 서포트 벡터 머신(SVM)에서 사용하는 힌지 손실(Hinge Loss)은 단순히 맞고 틀림을 넘어, 결정 경계에서 얼마나 멀리 또는 가깝게 떨어져 있는지도 고려하여 페널티를 부여합니다. 기하학적으로 손실 함수는 잘못 분류된 점들에 '벌점'을 주어, 결정 경계를 점차 올바른 위치로 '밀어내는' 역할을 합니다. 학습의 목표는 이 손실 함수의 값을 최소화하는 것입니다.

퀴즈: 선형 결정 경계는 w · x + b = 0으로 정의됩니다. 이때 가중치 벡터 w는 결정 경계(선)와 평행할까요? (O / X)

정답: X

해설: 벡터 w는 결정 경계의 법선 벡터(normal vector)로, 경계면에 항상 수직입니다. 벡터 w는 결정 경계가 어느 방향을 '바라보고' 있는지를 결정합니다. 어떤 점 x에 대해 w · x + b의 값이 양수이면 그 점은 w가 가리키는 방향(positive side)에 있고, 음수이면 반대 방향(negative side)에 있습니다. 따라서 w는 공간을 나누는 기준 방향 그 자체이며, 경계면과는 수직 관계를 이룹니다.

행렬과 선형 변환: 데이터 세상을 바꾸는 도구

벡터가 데이터를 나타내는 기본 단위라면, 행렬(Matrix)은 이 데이터를 능동적으로 변환하고 조작하는 강력한 도구입니다. 행렬을 통해 우리는 데이터의 차원을 바꾸거나, 특정 패턴을 부각시키거나, 여러 데이터를 한 번에 처리할 수 있습니다. 이 절에서는 행렬을 데이터의 '변환' 도구로 이해하고, 그 핵심 속성들이 모델 학습에 어떤 영향을 미치는지 탐구합니다.

1. 이론 1: 행렬과 선형 변환(Linear Transformation)

행렬(Matrix)은 숫자를 2차원 배열로 나열한 것이지만, 기하학적으로는 한 벡터를 다른 벡터로 매핑하는 '함수' 또는 '변환'으로 이해하는 것이 중요합니다. 특히, 행렬과 벡터의 곱셈 Ax는 입력 벡터 x를 행렬 A가 정의하는 새로운 위치로 이동시키는 선형 변환(Linear Transformation)을 의미합니다.

선형 변환은 두 가지 중요한 속성을 가집니다: (1) 원점은 항상 원점으로 유지된다. (2) 공간의 격자선(grid lines)은 변환 후에도 평행하고 등간격을 유지한다. 이러한 특성 덕분에 선형 변환은 공간을 일관성 있게 회전(rotation), 확대/축소(scaling), 비트는(shear) 등의 조합으로 표현할 수 있습니다.

2. 사례 연구: 2x2 변환으로 데이터 구름(Data Cloud) 변형시키기

행렬 변환의 효과를 시각적으로 이해하기 위해, 2차원 평면에 원형으로 분포된 데이터 포인트(데이터 구름)에 여러 변환 행렬을 적용해 보겠습니다. 아래 차트는 초기 데이터와 회전, 스케일 변환 후의 데이터 분포를 보여줍니다.

위 차트에서 볼 수 있듯이:

  • 회전 변환(Rotation): 회전 행렬 [[cosθ, -sinθ], [sinθ, cosθ]]를 곱하면 데이터 구름 전체가 원점을 중심으로 θ만큼 회전합니다. 데이터 포인트들 간의 상대적 거리나 형태는 변하지 않습니다.
  • 스케일 변환(Scaling): 대각 행렬 [[s₁, 0], [0, s₂]]를 곱하면 데이터가 x축 방향으로 s₁배, y축 방향으로 s₂배 늘어나거나 줄어듭니다. 원래 원형이었던 데이터 구름이 타원형으로 변하는 것을 볼 수 있습니다.

이러한 변환은 데이터 전처리 과정에서 특징들의 스케일을 맞추거나(Standardization), 주성분 분석(PCA)에서 데이터의 분산이 최대가 되는 새로운 축을 찾기 위해 데이터를 회전시키는 등 머신러닝의 여러 단계에서 핵심적인 역할을 합니다.

3. 이론 2: 행렬의 핵심 속성 - 랭크, 역행렬, 조건수

모든 행렬이 동일한 변환 능력을 갖는 것은 아닙니다. 행렬의 핵심 속성들은 그 변환의 특성을 결정합니다.

  • 랭크(Rank): 행렬 A에 의해 변환된 후의 공간이 몇 차원인지를 나타냅니다. 기하학적으로, 2x2 행렬의 랭크가 2이면 2차원 평면을 그대로 2차원 평면으로 변환하지만, 랭크가 1이면 2차원 평면을 1차원 직선으로 '압축'시킵니다. 랭크는 변환 후 정보가 얼마나 보존되는지를 알려주는 척도입니다.
  • 역행렬(Inverse Matrix): 변환 A를 '되돌리는' 변환을 A⁻¹로 표기합니다. 즉, A⁻¹(Ax) = x로 원래 벡터로 복원시킵니다. 역행렬은 행렬이 정사각형(square)이고 행렬식(determinant)이 0이 아닐 때만 존재합니다. 행렬식이 0이라는 것은 변환 과정에서 차원 축소(예: 평면 -> 선)가 일어나 정보 손실이 발생했음을 의미하며, 한번 압축된 정보는 완벽히 복원할 수 없으므로 역행렬이 존재하지 않습니다.
  • 조건수(Condition Number): 행렬이 역행렬을 구하기 어려운, 즉 '거의 특이 행렬(singular matrix)'에 가까운 정도를 나타내는 수치입니다. 조건수가 매우 크면 입력의 작은 오차가 출력에서 매우 큰 변화를 일으킬 수 있어, 모델 학습 과정이 수치적으로 불안정해지고 최적의 해를 찾기 어려워질 수 있습니다.

퀴즈: 행렬 A = [[2, 1], [4, 2]]의 역행렬은 존재하는가? 그 이유는 무엇인가?

정답: 존재하지 않는다.

해설: 이 행렬의 행렬식(determinant)은 det(A) = (2 * 2) - (4 * 1) = 0이기 때문입니다. 기하학적으로 이 변환은 2차원 공간의 모든 점을 y = 2x라는 하나의 직선 위로 사상(mapping)시킵니다. 두 번째 행 [4, 2]가 첫 번째 행 [2, 1]의 정확히 2배이므로, 두 행 벡터는 선형 종속 관계에 있어 독립적인 방향을 나타내지 못하고 하나의 방향으로 공간을 압축합니다. 이렇게 선으로 압축된 정보는 다시 원래의 2차원 평면으로 복원할 수 없으므로 역변환(역행렬)이 존재하지 않습니다.

고유값과 고유벡터: 데이터의 숨겨진 축을 찾아서

데이터를 변환하는 행렬의 효과를 더 깊이 이해하기 위한 열쇠는 고유값(Eigenvalue)과 고유벡터(Eigenvector)에 있습니다. 이 개념들은 행렬 변환의 '본질'을 드러내며, 데이터에 숨겨진 가장 중요한 구조적 축을 찾아내는 강력한 도구가 됩니다. 이는 주성분 분석(PCA)과 같은 차원 축소 기법의 핵심 원리입니다.

1. 이론 1: 고유값(Eigenvalue)과 고유벡터(Eigenvector)의 정의

정방 행렬 A에 대해, Ax = λx라는 관계를 만족하는 0이 아닌 벡터 x를 고유벡터, 스칼라 λ를 해당 고유벡터에 대한 고유값이라고 합니다.

이 수식의 기하학적 직관은 매우 중요합니다:

  • 고유벡터 (x): 행렬 A로 선형 변환을 적용했을 때, 다른 벡터들은 방향이 바뀌지만 고유벡터만은 방향이 변하지 않고 그대로 유지되는 특별한 벡터입니다.
  • 고유값 (λ): 해당 고유벡터 방향으로 변환이 얼마나 강력하게 일어나는지를 나타내는 스케일링 팩터(scaling factor)입니다. 고유값이 크면 그 방향으로 많이 늘어나고, 작으면 적게 늘어나거나 줄어듭니다.

즉, 고유벡터는 행렬 변환의 '축' 역할을 하며, 고유값은 그 축 방향의 '힘'을 나타냅니다. 데이터 분석에서는 데이터의 공분산 행렬에 대한 고유값-고유벡터 분석을 통해 데이터가 어떤 방향으로 가장 크게 분산되어 있는지를 파악할 수 있습니다.

2. 사례 연구: 타원형 데이터의 주축(Principal Axes) 찾기

두 변수 간에 강한 양의 상관관계가 있는 데이터가 있다고 가정해 봅시다. 이 데이터를 산점도로 그리면 오른쪽 위로 길쭉한 타원 형태의 '데이터 구름'이 나타날 것입니다. 이 데이터의 숨겨진 축, 즉 데이터의 분산을 가장 잘 설명하는 방향은 어떻게 찾을 수 있을까요?

  1. 공분산 행렬(Covariance Matrix) 계산: 먼저 데이터의 공분산 행렬 C를 계산합니다. 이 행렬은 데이터가 각 특징 축 방향으로 얼마나 퍼져있는지(분산), 그리고 특징들 간에 어떤 선형 관계가 있는지(공분산)를 담고 있는 대칭 행렬입니다.
  2. 고유값 분해(Eigendecomposition): 공분산 행렬 C의 고유값과 고유벡터를 계산합니다.

위 그림에서 볼 수 있듯이, 가장 큰 고유값(λ₁)에 해당하는 고유벡터(v₁, 그림의 PC1)는 데이터가 가장 길게 퍼져있는 방향, 즉 타원의 장축 방향과 정확히 일치합니다. 두 번째로 큰 고유값(λ₂)에 해당하는 고유벡터(v₂, 그림의 PC2)는 그에 수직인 단축 방향을 가리킵니다. 이 고유벡터들이 바로 데이터의 주성분(Principal Components)이며, 데이터의 분산을 가장 잘 설명하는 새로운 '좌표축'이 됩니다.

3. 이론 2: 행렬 분해와 SVD 개관

고유값과 고유벡터는 행렬을 더 근본적인 요소로 분해하는 강력한 개념으로 이어집니다.

  • 대칭 행렬의 고유 분해(Eigendecomposition): 스펙트럼 정리(Spectral Theorem)에 따르면, 모든 대칭 행렬 A는 A = PDPᵀ 형태로 분해할 수 있습니다. 여기서 P는 고유벡터들을 열로 갖는 직교 행렬(회전), D는 고유값들을 대각 원소로 갖는 대각 행렬(스케일링)입니다. 이는 모든 대칭 변환이 '회전 → 축 방향 스케일링 → 역회전'의 조합으로 이루어짐을 의미합니다.
  • 특이값 분해(Singular Value Decomposition, SVD): 고유 분해는 정방 행렬에만 적용 가능하지만, SVD는 이를 모든 m x n 행렬로 일반화한 것입니다. SVD는 모든 행렬 A를 A = UΣVᵀ로 분해합니다. 여기서 U와 V는 직교 행렬(회전), Σ는 특이값(singular values)을 대각 원소로 갖는 대각 행렬(스케일링)입니다. SVD는 모든 선형 변환을 ';입력 공간에서의 회전(Vᵀ) → 스케일링(Σ) → 출력 공간에서의 회전(U)'이라는 세 단계로 분해하는 가장 일반적이고 강력한 도구입니다. PCA, 추천 시스템, 자연어 처리(LSA) 등 머신러닝 전반에서 핵심적인 역할을 수행합니다.

퀴즈: 어떤 2차원 데이터의 공분산 행렬이 C = [[5, 3], [3, 5]]일 때, 이 데이터의 분산이 가장 큰 방향(제1 주성분)을 나타내는 벡터는 다음 중 무엇일까요? (1) [1, 0] (2) [1, -1] (3) [1, 1]

정답: (3) [1, 1]

해설: 제1 주성분은 가장 큰 고유값에 해당하는 고유벡터입니다. 각 후보 벡터를 공분산 행렬 C에 곱하여 Cx = λx 관계가 성립하는지 확인합니다.

  • x = [1, 1]ᵀ 일 때: C x = [[5, 3], [3, 5]] [1, 1]ᵀ = [8, 8]ᵀ = 8 * [1, 1]ᵀ. 따라서 고유값 λ₁ = 8.
  • x = [1, -1]ᵀ 일 때: C x = [[5, 3], [3, 5]] [1, -1]ᵀ = [2, -2]ᵀ = 2 * [1, -1]ᵀ. 따라서 고유값 λ₂ = 2.

더 큰 고유값은 8이므로, 이에 해당하는 고유벡터 [1, 1]이 데이터의 분산이 가장 큰 방향, 즉 제1 주성분(PC1)이 됩니다.
