도입: 예측의 '좋고 나쁨'을 어떻게 측정할까?

이번 챕터에서는 머신러닝 모델, 특히 확률을 출력하는 분류 모델의 성능을 어떻게 정량적으로 평가하고 최적화하는지 학습합니다. 이를 위해 정보이론의 핵심 개념들을 빌려와 '손실 함수(Loss Function)'를 이해하는 여정을 시작합니다. 손실 함수는 모델의 예측이 실제 값과 얼마나 다른지를 나타내는 척도로, 모델 학습의 나침반 역할을 합니다. DataCamp의 설명에 따르면, 손실 함수는 모델의 성능과 정확성을 측정 가능한 방식으로 제공하여 학습 과정을 안내합니다.

Lecture 1에서 배운 확률의 개념을 바탕으로, 이제 우리는 한 단계 더 나아가 모델이 예측한 '확률 분포'가 실제 데이터의 '확률 분포'와 얼마나 다른지, 즉 예측의 '오차'를 측정하는 방법을 탐구할 것입니다. 이 오차를 최소화하는 과정이 바로 머신러닝 모델의 '학습'입니다.

핵심 질문: "모델 A는 내일 비가 올 확률을 80%로, 모델 B는 60%로 예측했습니다. 실제로 비가 왔을 때, 어떤 모델이 얼마나 더 '좋은' 예측을 한 것일까요?"

이 질문에 답하기 위해서는 직관을 넘어선 체계적인 평가 도구가 필요합니다. 정보이론은 바로 이 '예측의 좋고 나쁨'을 수학적으로 정의하고 측정하는 강력한 프레임워크를 제공하며, 이는 현대 머신러닝에서 사용되는 손실 함수의 근간을 이룹니다. 이번 챕터를 통해 우리는 이 질문에 대한 명확한 답을 찾는 방법을 배우게 될 것입니다.

정보량의 측정: 엔트로피와 교차 엔트로피

이 파트에서는 정보의 양을 측정하는 기본 단위부터 시작하여, 두 확률 분포 간의 차이를 측정하는 교차 엔트로피의 개념을 학습합니다. 그리고 이것이 왜 분류 모델의 손실 함수로 사용되는지 그 본질을 파헤칩니다.

사례 (Case Study): 날씨 예측 모델 비교하기

내일의 날씨가 '비(Rain)' 또는 '맑음(Clear)' 두 가지 경우만 있다고 가정해 봅시다. 그리고 실제 관측 결과, 내일은 '비'가 왔습니다. 두 명의 예보관(모델)이 다음과 같이 예측했습니다.

  • 모델 A (초보 예보관): 비가 올 확률 P(Rain|A) = 0.6, 맑을 확률 P(Clear|A) = 0.4
  • 모델 B (전문 예보관): 비가 올 확률 P(Rain|B) = 0.9, 맑을 확률 P(Clear|B) = 0.1

직관적으로 실제 결과인 '비'에 더 높은 확률을 부여한 모델 B가 더 나은 예측을 한 것으로 보입니다. 하지만 이 '더 나음'을 어떻게 수치로 표현하고, 두 모델의 예측 품질 차이를 정량적으로 비교할 수 있을까요? 이 질문에 답하기 위해, 우리는 먼저 '정보'를 측정하는 방법부터 알아봐야 합니다. 이 필요성이 바로 교차 엔트로피 개념으로 이어집니다.

이론 (Theory): 정보량, 엔트로피, 그리고 교차 엔트로피의 정의

1. 정보량 (Information Content / Self-Information)

정보이론의 창시자 클로드 섀넌(Claude Shannon)은 정보량을 '놀라움의 정도(degree of surprise)'로 정의했습니다. 드물게 발생하는 사건(확률이 낮은 사건)일수록, 그 사건이 실제로 발생했다는 것을 알게 되었을 때 우리는 더 많이 놀라게 되며, 이는 더 많은 정보량을 얻었음을 의미합니다. 예를 들어, '해가 동쪽에서 떴다'는 소식보다 '해가 서쪽에서 떴다'는 소식이 훨씬 더 큰 정보량을 가집니다.

어떤 사건 x가 발생할 확률이 P(x)일 때, 이 사건의 정보량 I(x)는 다음과 같이 정의됩니다.

I(x) = -log(P(x))

로그 함수의 밑(base)은 정보량의 단위를 결정합니다. 밑이 2이면 비트(bit), 자연상수 e이면 내트(nat)를 사용합니다. 확률 P(x)가 1에 가까울수록 정보량은 0에 수렴하고, P(x)가 0에 가까울수록 정보량은 무한대로 발산합니다. 이는 사건이 예측 가능할수록 새로운 정보가 없다는 직관과 일치합니다.

2. 엔트로피 (Entropy)

엔트로피는 어떤 확률 분포 P가 가지는 불확실성의 기댓값, 즉 평균 정보량을 의미합니다. 확률 변수 X가 가질 수 있는 모든 사건 x에 대해, 각 사건의 정보량과 그 사건이 일어날 확률을 곱한 값을 모두 더하여 계산합니다.

H(P) = Ex~P[I(x)] = - ∑x P(x)log(P(x))

엔트로피는 분포의 '무질서도' 또는 '예측하기 어려움'의 정도로 해석할 수 있습니다. 예를 들어, 앞면과 뒷면이 나올 확률이 각각 0.5인 공정한 동전은, 던지기 전까지 결과를 예측하기 가장 어렵기 때문에 엔트로피가 최대가 됩니다. 반면, 앞면만 나오는 동전은 결과가 100% 확실하므로 엔트로피가 0입니다.

3. 교차 엔트로피 (Cross-Entropy)

교차 엔트로피는 정보이론과 머신러닝을 잇는 핵심적인 다리입니다. 실제 확률 분포가 P인 사건들에 대해, 우리가 만든 모델이 예측한 확률 분포 Q를 사용하여 정보량을 계산했을 때의 평균 정보량을 의미합니다. 즉, '잘못된' 예측 분포 Q를 가지고 실제 사건들을 얼마나 잘 설명하는지를 측정하는 것입니다.

H(P, Q) = Ex~P[-log(Q(x))] = - ∑x P(x)log(Q(x))

만약 모델의 예측 분포 Q가 실제 분포 P와 완벽하게 동일하다면, 교차 엔트로피는 실제 분포의 엔트로피 H(P)와 같아집니다. 하지만 Q가 P와 다를수록, 교차 엔트로피 값은 H(P)보다 커지게 됩니다. 이 '추가적인' 정보량이 바로 모델의 예측 오차를 나타냅니다.

핵심 연결고리: 교차 엔트로피와 음의 로그 우도(Negative Log-Likelihood)

분류 문제에서 교차 엔트로피가 손실 함수로 사용되는 이유를 이해하는 것은 매우 중요합니다. 이 관계는 최대우도추정(Maximum Likelihood Estimation, MLE)과 깊이 연결됩니다.

  1. 실제 정답(P)의 표현: 분류 문제에서 실제 정답 레이블은 특정 클래스 하나만 1이고 나머지는 0인 '원-핫(one-hot)' 벡터로 표현할 수 있습니다. 이는 해당 클래스가 발생할 확률이 100%인 확률 분포 P로 볼 수 있습니다. 예를 들어, '고양이' 이미지는 [고양이=1, 개=0, 새=0]으로 표현됩니다.
  2. 모델 예측(Q)의 표현: 모델은 각 클래스에 대한 확률을 출력합니다 (예: Softmax 함수를 통해). 이는 모델이 예측한 확률 분포 Q입니다. 예를 들어, [고양이=0.7, 개=0.2, 새=0.1]과 같습니다.
  3. 교차 엔트로피 계산: 이 두 분포 P와 Q에 대해 교차 엔트로피를 계산해 봅시다.
    H(P, Q) = - ∑i=1C P(classi) * log(Q(classi))
    P는 원-핫 벡터이므로 정답 클래스 k에 대해서만 P(class_k)=1이고 나머지는 0입니다. 따라서 위 식은 다음과 같이 단순화됩니다.
    H(P, Q) = - (1 * log(Q(classk)) + 0 * ... ) = -log(Q(classk))
  4. 음의 로그 우도(NLL): 한편, 주어진 데이터에 대해 모델의 우도(Likelihood)는 모델이 실제 정답을 맞힐 확률, 즉 Q(class_k)입니다. 최대우도추정(MLE)은 이 우도를 최대화하는 것을 목표로 합니다. 로그 변환은 단조 증가 함수이므로, 우도를 최대화하는 것은 로그 우도 log(Q(class_k))를 최대화하는 것과 같습니다. 그리고 최적화 문제는 보통 '최소화' 문제로 다루므로, 이는 음의 로그 우도 -log(Q(class_k))를 최소화하는 것과 동일합니다.

결론적으로, 분류 모델에서 교차 엔트로피 손실을 최소화하는 것은 모델의 로그 우도를 최대화하는 것과 수학적으로 동일한 목표를 가집니다. 이것이 교차 엔트로피가 분류 문제의 표준 손실 함수로 자리 잡은 근본적인 이유입니다.

예제 (Example): 날씨 예측 모델의 교차 엔트로피 계산

문제

앞선 사례에서 제시된 모델 A와 모델 B의 교차 엔트로피 손실을 각각 계산하고, 어느 모델의 손실이 더 낮은지 비교하시오. (실제 정답 분포 P = [Rain=1, Clear=0])

풀이 과정

교차 엔트로피 공식 H(P, Q) = - Σ P(x)log(Q(x))를 적용합니다. 여기서 P는 실제 분포, Q는 모델의 예측 분포입니다.

  • 모델 A의 손실 (QA = [0.6, 0.4]):
    LA = H(P, QA) = - [1 * log(0.6) + 0 * log(0.4)] = -log(0.6) ≈ 0.5108
  • 모델 B의 손실 (QB = [0.9, 0.1]):
    LB = H(P, QB) = - [1 * log(0.9) + 0 * log(0.1)] = -log(0.9) ≈ 0.1054

결론

모델 B의 손실 값(약 0.1054)이 모델 A의 손실 값(약 0.5108)보다 현저히 낮습니다. 이는 모델 B가 실제 결과를 더 높은 확률로 예측했기 때문이며, 교차 엔트로피를 통해 이 '더 나음'을 정량적으로 평가할 수 있음을 보여줍니다. 머신러닝 모델은 학습 과정에서 이 손실 값을 최소화하는 방향으로 파라미터를 조정하게 됩니다.

두 분포의 거리: 쿨백-라이블러 발산 (Kullback-Leibler Divergence)

교차 엔트로피가 손실 함수로 사용되는 이유를 더 깊이 이해하기 위해, 두 확률 분포 사이의 '거리' 개념인 쿨백-라이블러 발산(Kullback-Leibler Divergence, KLD)을 탐구합니다. KLD는 한 확률 분포를 다른 확률 분포로 근사할 때 발생하는 정보 손실량을 측정합니다.

사례 (Case Study): 스팸 메일 필터의 성능 평가

우리가 스팸 메일 필터를 개발한다고 상상해 봅시다. 이 필터의 성능은 정상 메일과 스팸 메일을 얼마나 잘 구분하는지에 달려 있습니다.

  • P (실제 분포): 실제 세상의 모든 정상 메일에서 '광고', '무료', '당첨'과 같은 특정 단어들이 나타나는 실제 확률 분포입니다. 이는 우리가 완벽히 알 수는 없지만 존재하는 '참 분포'입니다.
  • Q (모델의 예측 분포): 우리가 가진 데이터로 학습한 스팸 필터 모델이 추정한, 정상 메일에서의 단어 확률 분포입니다.

여기서 핵심 질문은 "우리 모델(Q)이 실제(P)를 얼마나 잘못 이해하고 있는가?" 입니다. 이 두 분포 간의 '차이' 또는 모델이 P를 Q로 근사하면서 잃어버린 '정보 손실량'을 어떻게 측정할 수 있을까요? 이 질문에 대한 답이 바로 KL 발산입니다. KL 발산은 모델의 추정 분포 Q가 실제 분포 P를 얼마나 잘 근사하는지 측정하는 데 사용될 수 있습니다.

이론 (Theory): KL 발산의 정의와 특성

1. KL 발산(KLD)의 정의

쿨백-라이블러 발산은 두 확률 분포 P와 Q에 대해 정의되며, 분포 Q를 사용하여 분포 P의 정보를 인코딩할 때 발생하는 평균적인 추가 정보량(비트 수)을 의미합니다. 즉, P를 기준으로 할 때 Q가 얼마나 비효율적인지를 나타냅니다. '상대적 엔트로피(Relative Entropy)'라고도 불립니다.

DKL(P || Q) = ∑x P(x) log( P(x) / Q(x) )

이 식은 각 사건 x에 대해 P(x)와 Q(x)의 확률 비율에 로그를 취한 값의 기댓값으로 해석할 수 있습니다. 만약 모든 x에 대해 P(x) = Q(x)라면, 로그 안의 값은 1이 되어 log(1) = 0이므로, KLD는 0이 됩니다.

2. KLD와 교차 엔트로피의 관계 (핵심)

KLD의 정의를 로그의 성질을 이용해 분해하면 교차 엔트로피와의 명확한 관계를 도출할 수 있습니다. 이는 머신러닝에서 교차 엔트로피를 손실 함수로 사용하는 이론적 기반이 됩니다.

DKL(P || Q) = ∑ P(x) [log(P(x)) - log(Q(x))]
= ∑ P(x)log(P(x)) - ∑ P(x)log(Q(x))
= -H(P) + H(P, Q)

따라서, 우리는 다음과 같은 매우 중요한 관계식을 얻습니다.

H(P, Q) = DKL(P || Q) + H(P)

교차 엔트로피 = KL 발산 + 실제 분포의 엔트로피

머신러닝 모델을 학습시킬 때, 우리의 목표는 모델의 예측 분포 Q를 실제 데이터의 분포 P에 가깝게 만드는 것입니다. 여기서 실제 데이터의 분포 P는 주어져 있으며 변하지 않으므로, 그 엔트로피 H(P)는 상수 값입니다. 따라서, 교차 엔트로피 H(P, Q)를 최소화하는 것은 KL 발산 D_KL(P || Q)를 최소화하는 것과 완벽하게 동일한 최적화 목표를 가집니다. 이것이 바로 우리가 계산이 더 간단한 교차 엔트로피를 손실 함수로 사용하는 근본적인 이유입니다.

3. KLD의 주요 특성

  • 비대칭성 (Asymmetry): D_KL(P || Q) ≠ D_KL(Q || P) 입니다. 이는 KLD가 수학적 의미의 엄격한 '거리(distance metric)'가 아님을 의미합니다. D_KL(P || Q)는 P를 기준으로 Q의 차이를 측정하는 반면, D_KL(Q || P)는 Q를 기준으로 P의 차이를 측정합니다. 최적화 관점에서 이는 다른 결과를 낳을 수 있습니다. 예를 들어, 실제 분포 P가 여러 개의 봉우리(multi-modal)를 가질 때, D_KL(P || Q)를 최소화하는 모델 Q는 모든 봉우리를 어중간하게 포함하려는 경향(평균을 찾는 경향)을 보이고, D_KL(Q || P)를 최소화하는 모델 Q는 여러 봉우리 중 하나에 집중하려는 경향(하나의 모드를 찾는 경향)을 보입니다.
  • 음이 아닌 값 (Non-negativity): 깁스 부등식(Gibbs' inequality)에 의해 D_KL(P || Q) ≥ 0 이 항상 성립합니다. 두 분포가 완전히 동일할 때만 KLD 값은 0이 됩니다.

예제 (Example): 두 베르누이 분포의 KLD 계산

문제

동전 던지기에서 앞면이 나올 실제 확률 분포 P는 P(앞면)=0.8, P(뒷면)=0.2인 베르누이 분포를 따른다고 합니다. 한 모델이 예측한 확률 분포 Q는 Q(앞면)=0.5, Q(뒷면)=0.5인 베르누이 분포입니다. 이 때, D_KL(P || Q)를 계산하시오. (로그의 밑은 자연로그 e를 사용)

풀이 과정

KLD 공식에 따라 각 사건(앞면, 뒷면)에 대한 값을 계산하여 더합니다.

DKL(P || Q) = P(앞면) * log( P(앞면) / Q(앞면) ) + P(뒷면) * log( P(뒷면) / Q(뒷면) )

값을 대입하면 다음과 같습니다.

= 0.8 * log(0.8 / 0.5) + 0.2 * log(0.2 / 0.5)
= 0.8 * log(1.6) + 0.2 * log(0.4)
≈ 0.8 * (0.470) + 0.2 * (-0.916)
≈ 0.376 - 0.1832 = 0.1928

결론

모델의 예측 분포 Q를 사용하여 실제 분포 P를 표현할 때 발생하는 평균 정보 손실량은 약 0.1928 내트(nats)입니다. 이 값이 0보다 크다는 것은 두 분포가 다르다는 것을 의미하며, 모델 학습은 이 값을 0에 가깝게 만드는 과정이라고 할 수 있습니다.

정보이론으로 해석하는 로지스틱 회귀

이제 앞서 배운 정보이론의 개념들을 로지스틱 회귀(Logistic Regression)라는 구체적인 모델에 적용해 보겠습니다. 이를 통해 모델의 학습 과정이 정보이론적 관점에서 어떻게 해석될 수 있는지 통합적으로 이해할 수 있습니다.

사례 (Case Study): 로지스틱 회귀 학습 과정의 재해석

날씨 데이터(온도, 습도 등)를 입력받아 내일 비가 올지(1) 안 올지(0)를 예측하는 로지스틱 회귀 모델의 학습 과정을 정보이론적 관점에서 단계별로 다시 살펴보겠습니다.

  1. 확률 예측: 모델은 입력 데이터 x와 가중치 w를 선형 결합한 후, 시그모이드(Sigmoid) 함수를 통과시켜 '비가 올 확률' p = σ(wᵀx)를 출력합니다. 이는 모델이 예측한 베르누이 확률 분포 Q입니다.
  2. 손실 계산: 이 예측 확률 p와 실제 결과 y (1 또는 0) 사이의 교차 엔트로피 손실(Cross-Entropy Loss)을 계산합니다. 이 손실은 모델 예측 분포 Q와 실제 데이터 분포 P(y=1일 확률이 100% 또는 0%인 분포) 사이의 차이를 나타냅니다.
  3. 가중치 업데이트: 경사 하강법(Gradient Descent)을 이용해 이 손실을 최소화하는 방향으로 모델의 가중치 w를 업데이트합니다.
  4. 정보이론적 해석: 이 전체 과정은 결국 모델의 예측 분포 Q가 실제 데이터의 분포 P와의 KL 발산을 최소화하는 과정과 수학적으로 동일합니다. 즉, 로지스틱 회귀의 학습은 정보 손실을 줄여나가는 과정으로 재해석될 수 있습니다.

실제 강우 예측 연구에서도 로지스틱 회귀는 주어진 기상 데이터를 바탕으로 비가 올 확률을 모델링하는 데 효과적으로 사용됩니다.

이론 (Theory): 우도, 에너지, 그리고 사전분포의 연결

1. 로지스틱 회귀와 우도 (Likelihood)

로지스틱 회귀의 학습 과정을 이해하는 핵심은 최대우도추정(MLE)입니다. 모델의 목표는 주어진 데이터를 가장 잘 설명하는, 즉 데이터가 관측될 확률(우도)을 최대로 만드는 파라미터를 찾는 것입니다.

  • 베르누이 분포로서의 출력: 로지스틱 회귀의 출력 p는 각 데이터 포인트에 대한 베르누이 분포의 파라미터(성공 확률)로 해석할 수 있습니다. 실제 레이블이 y (0 또는 1)일 때, 해당 데이터 포인트의 확률은 p^y * (1-p)^(1-y)로 표현됩니다.
  • 우도 함수: 전체 데이터셋에 대한 우도 함수 L(w)는 모든 데이터 포인트의 확률들의 곱으로 정의됩니다 (데이터가 독립적이라고 가정).
    • L(w) = ∏i=1N piyi (1-pi)(1-yi)
  • 로그 우도와 교차 엔트로피: 우도 함수에 로그를 취하면 곱셈이 덧셈으로 바뀌어 계산이 용이해집니다. 이를 로그 우도(Log-Likelihood)라고 합니다.
    log L(w) = ∑i=1N [yi log(pi) + (1-yi) log(1-pi)]
    이 식에 음수를 취하고 평균을 내면, - (1/N) * log L(w), 이는 정확히 이진 교차 엔트로피(Binary Cross-Entropy) 손실 함수의 정의와 일치합니다. 따라서, 교차 엔트로피 손실을 최소화하는 것은 우도를 최대화하는 것과 같습니다.

2. 정규화(Regularization)와 사전 분포(Prior Distribution)

과적합을 방지하기 위해 사용되는 정규화는 베이지안 통계학의 관점에서 흥미롭게 해석될 수 있습니다. 이는 모델 파라미터에 대한 우리의 '사전 믿음(prior belief)'을 반영하는 과정입니다.

  • MAP (Maximum A Posteriori) 추정: 베이즈 정리에 따르면, 사후 확률은 우도와 사전 확률의 곱에 비례합니다: P(w|D) ∝ P(D|w) * P(w). MAP는 이 사후 확률을 최대화하는 파라미터 w를 찾는 것을 목표로 합니다.
  • L2 정규화와 가우시안 사전 분포: 손실 함수에 추가되는 L2 정규화 항(λ||w||²)은 모델의 가중치(w)가 평균이 0이고 분산이 특정 값인 가우시안 분포를 따른다는 사전 믿음을 부여하는 것과 수학적으로 동일합니다. 즉, 가중치 값들이 0에 가깝게 분포할 것이라고 가정하는 것입니다. 이는 가중치 값이 비정상적으로 커지는 것을 방지하여 모델을 더 일반화시키는 효과를 낳습니다.
  • 결론: 따라서 정규화는 단순히 과적합을 막는 기술적 트릭을 넘어, 모델 파라미터에 대한 확률론적 가정을 추가하는 베이지안적 해석이 가능합니다. 이는 MLE(빈도주의적 관점)를 MAP(베이지안적 관점)로 확장하는 것으로 볼 수 있습니다.

예제 (Example): 로짓(Logit)과 확률(Probability)의 상호 변환

문제 1

로지스틱 회귀 모델의 선형 결합 결과(로짓)가 z = 1.5로 계산되었습니다. 이 때, 해당 샘플이 클래스 1에 속할 확률은 얼마인가요?

풀이

로짓 값을 확률로 변환하기 위해 시그모이드 함수를 사용합니다.

p = σ(z) = 1 / (1 + e-z)
p = 1 / (1 + e-1.5) ≈ 1 / (1 + 0.2231) ≈ 0.8176

따라서 해당 샘플이 클래스 1에 속할 확률은 약 81.76%입니다.

문제 2

어떤 샘플이 클래스 1에 속할 확률을 90%(p=0.9)로 만들고 싶습니다. 모델의 로짓 값은 얼마가 되어야 하나요?

풀이

확률 값을 로짓으로 변환하기 위해 로짓 함수(로그 오즈)를 사용합니다.

z = logit(p) = log(p / (1-p))
z = log(0.9 / (1-0.9)) = log(0.9 / 0.1) = log(9) ≈ 2.197

모델의 로짓 값은 약 2.197이 되어야 합니다.

이 예제들은 로지스틱 회귀 모델 내부에서 선형적인 로짓 값과 비선형적인 확률 값이 어떻게 상호 변환되는지 보여주며, 모델의 작동 방식에 대한 구체적인 이해를 돕습니다. AWS의 설명에 따르면, 로짓 함수는 y를 x의 시그모이드 함수로 매핑합니다.

핵심 내용 정리

  • 정보이론은 불확실성을 정량화하는 수학적 도구이며, 정보량, 엔트로피, 교차 엔트로피, KL 발산이 핵심 개념입니다.
  • 분류 문제의 손실 함수로 널리 쓰이는 교차 엔트로피(Cross-Entropy)는 수학적으로 음의 로그 우도(Negative Log-Likelihood)와 동일하며, 이를 최소화하는 것은 모델의 우도를 최대화하는 것과 같습니다.
  • 교차 엔트로피를 최소화하는 것은 실제 데이터 분포와 모델 예측 분포 간의 KL 발산(Kullback-Leibler Divergence)을 최소화하는 것과 동일한 목표를 가집니다. 이는 모델 학습의 정보이론적 본질입니다.
  • 로지스틱 회귀의 학습은 결국 모델의 예측 분포와 실제 데이터 분포 간의 KL 발산을 줄여나가는 과정으로 해석할 수 있습니다.
  • 정규화(Regularization)는 과적합을 방지하는 기술을 넘어, 베이지안 관점에서 모델 파라미터에 대한 사전 분포(Prior Distribution)를 가정하는 것과 같습니다. (예: L2 정규화 ↔ 가우시안 사전 분포)

이번 챕터에서 다룬 개념들은 이진 분류를 중심으로 설명되었지만, 다중 클래스 분류 문제로 자연스럽게 확장됩니다. 이 경우 시그모이드 함수 대신 소프트맥스(Softmax) 함수를 사용하고, 손실 함수로는 범주형 교차 엔트로피(Categorical Cross-Entropy)를 사용하게 됩니다. 원리는 동일하게 적용됩니다.

다음 챕터에서는 이렇게 정의된 손실 함수를 실제로 어떻게 최소화하는지, 즉 최적의 모델 파라미터를 찾아가는 '최적화(Optimization)'의 세계로 떠나보겠습니다. 가장 기본적인 경사 하강법의 원리부터 시작하여, 학습 속도와 안정성을 개선하기 위한 다양한 심화 기법들까지 다룰 것입니다.
