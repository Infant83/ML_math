# 정보이론과 손실함수

머신러닝, 특히 분류(Classification) 문제에서 모델의 성능을 평가하고 최적화하는 데 있어 **정보이론(Information Theory)**은 핵심적인 수학적 기반을 제공합니다. 정보이론은 정보의 양을 정량화하고, 불확실성을 측정하며, 두 확률 분포 간의 유사성 또는 차이를 비교하는 도구들을 제공합니다. 본 장에서는 정보량, 엔트로피, 교차 엔트로피, 로그 우도, 그리고 Kullback-Leibler Divergence(KLD)의 개념을 깊이 있게 탐구하고, 이들이 머신러닝 모델의 손실 함수로 어떻게 활용되는지 정보이론적 관점에서 해석합니다.

## 1. 정보량, 엔트로피, 교차 엔트로피, 로그 우도

### 1.1. 정보량 (Information Content)

**직관적 이해**: 어떤 사건이 발생했을 때 우리가 얻는 '놀라움'의 정도를 수치화한 것입니다. 발생할 확률이 낮은 사건일수록 발생했을 때 더 많은 정보를 제공하고, 놀라움의 정도가 큽니다. 예를 들어, "내일 해가 뜬다"는 정보는 거의 놀랍지 않지만, "내일 서울에 눈이 온다"는 정보는 여름이라면 매우 놀랍습니다.

**수학적 정의**: 사건 $x$가 발생할 확률이 $P(x)$일 때, 사건 $x$의 정보량 $I(x)$는 다음과 같이 정의됩니다.

$$ I(x) = -\log_2 P(x) $$

*   로그의 밑으로 2를 사용하는 것은 정보량을 '비트(bit)' 단위로 측정하기 위함입니다.
*   $P(x)=1$ (확실한 사건)일 때 $I(x)=0$ (정보량 없음).
*   $P(x) \to 0$ (매우 희귀한 사건)일 때 $I(x) \to \infty$ (매우 많은 정보량).

**예시**: 공정한 동전 던지기에서 앞면이 나올 확률은 $P(\text{앞면}) = 0.5$입니다. 이때 앞면이 나왔을 때 얻는 정보량은 $I(\text{앞면}) = -\log_2(0.5) = 1$ 비트입니다. 이는 2가지 경우의 수를 표현하는 데 1비트가 필요하다는 의미와 일치합니다.

### 1.2. 엔트로피 (Entropy)

**직관적 이해**: 확률 변수가 가질 수 있는 모든 결과에 대한 '평균 정보량' 또는 '불확실성'의 정도를 나타냅니다. 엔트로피가 높을수록 예측하기 어렵고 불확실성이 크며, 엔트로피가 낮을수록 예측하기 쉽고 불확실성이 작습니다.

**수학적 정의**: 확률 분포 $P$를 따르는 이산 확률 변수 $X$의 엔트로피 $H(P)$는 각 사건의 정보량에 해당 사건의 확률을 곱하여 모두 더한 값입니다.

$$ H(P) = E_{P}[I(x)] = -\sum_{x} P(x) \log_2 P(x) $$

*   엔트로피는 항상 0 이상입니다.
*   확률 분포가 균일할수록(모든 사건의 발생 확률이 비슷할수록) 엔트로피는 최대가 됩니다. 이는 예측 불가능성이 가장 높다는 의미입니다.

**예시**: 공정한 동전(앞면 0.5, 뒷면 0.5)의 엔트로피는 $H(P) = -(0.5 \log_2 0.5 + 0.5 \log_2 0.5) = -(0.5 \times -1 + 0.5 \times -1) = -(-0.5 - 0.5) = 1$ 비트입니다. 반면, 항상 앞면만 나오는 동전(앞면 1.0, 뒷면 0.0)의 엔트로피는 $H(P) = -(1.0 \log_2 1.0 + 0.0 \log_2 0.0) = -(1.0 \times 0 + 0) = 0$ 비트입니다. (여기서 $0 \log_2 0$은 0으로 간주합니다.)

### 1.3. 교차 엔트로피 (Cross Entropy)

**직관적 이해**: 두 확률 분포 $P$ (실제 분포)와 $Q$ (모델 예측 분포)가 얼마나 다른지를 측정하는 척도입니다. 실제 분포 $P$를 모델 $Q$를 사용하여 인코딩할 때 필요한 '평균 비트 수'로 해석할 수 있습니다. 모델 $Q$가 실제 분포 $P$를 잘 모방할수록 교차 엔트로피 값은 낮아집니다.

**수학적 정의**: 실제 확률 분포 $P$와 모델이 예측한 확률 분포 $Q$ 사이의 교차 엔트로피 $H(P, Q)$는 다음과 같습니다.

$$ H(P, Q) = -\sum_{x} P(x) \log_2 Q(x) $$

*   머신러닝에서는 주로 분류 모델의 손실 함수로 사용됩니다. 모델의 예측 $Q$가 실제 정답 $P$와 얼마나 일치하는지 정량화하며, 이 값을 최소화하는 방향으로 모델을 학습시킵니다.
*   $P(x)$는 실제 정답이므로, 실제 정답 클래스에 해당하는 $Q(x)$ 값만 손실 계산에 기여합니다. 즉, 모델이 정답 클래스에 부여한 확률이 낮을수록 손실이 커집니다.

**날씨 예측 확률의 교차 엔트로피 비교 사례**:

내일 비가 올지(Rain) 맑을지(Clear)를 예측하는 모델을 가정해 봅시다.

| 날씨 | 실제 확률 $P$ (Rain, Clear) | 모델 1 예측 $Q_1$ (Rain, Clear) | 모델 2 예측 $Q_2$ (Rain, Clear) |
|---|---|---|---|
| 실제: 비 | (1.0, 0.0) | (0.8, 0.2) | (0.3, 0.7) |
| 실제: 맑음 | (0.0, 1.0) | (0.1, 0.9) | (0.6, 0.4) |

**교차 엔트로피 계산**:

1.  **실제 '비'인 경우**:
    *   모델 1: $H(P, Q_1) = -(1.0 \log_2 0.8 + 0.0 \log_2 0.2) = -\log_2 0.8 \approx 0.32$
    *   모델 2: $H(P, Q_2) = -(1.0 \log_2 0.3 + 0.0 \log_2 0.7) = -\log_2 0.3 \approx 1.74$
    (모델 1이 실제 '비'에 더 높은 확률을 부여했으므로 손실이 더 작습니다.)

2.  **실제 '맑음'인 경우**:
    *   모델 1: $H(P, Q_1) = -(0.0 \log_2 0.1 + 1.0 \log_2 0.9) = -\log_2 0.9 \approx 0.15$
    *   모델 2: $H(P, Q_2) = -(0.0 \log_2 0.6 + 1.0 \log_2 0.4) = -\log_2 0.4 \approx 1.32$
    (모델 1이 실제 '맑음'에 더 높은 확률을 부여했으므로 손실이 더 작습니다.)

이처럼 교차 엔트로피는 모델이 정답 클래스에 얼마나 높은 확률을 부여했는지에 따라 손실을 계산합니다. 모델 1이 모델 2보다 실제 확률 분포를 더 잘 예측했으므로 교차 엔트로피 값이 더 낮게 나옵니다.

### 1.4. 로그 우도 (Log-Likelihood)

**개념**: 주어진 모델 파라미터 하에서 관측된 데이터가 나타날 확률(우도)에 로그를 취한 값입니다. 머신러닝 모델을 학습시키는 것은 이 로그 우도 값을 최대화하는 파라미터를 찾는 과정과 동일합니다.

**수학적 정의**: 데이터셋 $D = \{(x_1, y_1), \dots, (x_N, y_N)\}$과 모델 파라미터 $\theta$가 주어졌을 때, 로그 우도 $L(\theta|D)$는 다음과 같습니다.

$$ L(\theta|D) = \sum_{i=1}^{N} \log P(y_i|x_i, \theta) $$

*   여기서 $P(y_i|x_i, \theta)$는 모델이 입력 $x_i$에 대해 실제 라벨 $y_i$를 예측할 확률입니다.
*   로그를 취하는 이유는 곱셈 형태의 우도 함수를 덧셈 형태로 변환하여 계산을 용이하게 하고, 언더플로우(underflow)를 방지하기 위함입니다.

**교차 엔트로피와의 관계**: 분류 문제에서 로그 우도를 최대화하는 것은 **음의 로그 우도(Negative Log-Likelihood, NLL)**를 최소화하는 것과 같습니다. 이 NLL은 이진 분류의 경우 이진 교차 엔트로피(Binary Cross-Entropy)와, 다중 클래스 분류의 경우 범주형 교차 엔트로피(Categorical Cross-Entropy)와 동일합니다.

$$ \text{Minimize NLL} \iff \text{Maximize Log-Likelihood} $$
$$ \text{NLL} = -\frac{1}{N} \sum_{i=1}^{N} \log P(y_i|x_i, \theta) $$

따라서, 교차 엔트로피를 손실 함수로 사용하여 모델을 학습시키는 것은 통계적으로 최대 우도 추정(Maximum Likelihood Estimation, MLE)을 수행하는 것과 동일한 의미를 가집니다.

## 2. Kullback-Leibler Divergence (KLD)

### 2.1. KLD의 정의

**직관적 이해**: KLD는 두 확률 분포 $P$와 $Q$가 얼마나 '다른지'를 측정하는 척도입니다. $P$를 '참 분포'로, $Q$를 '근사 분포' 또는 '모델 분포'로 생각할 때, KLD는 $Q$가 $P$를 얼마나 잘 근사하는지, 또는 $Q$를 사용하여 $P$를 표현할 때 발생하는 '정보 손실'의 양을 나타냅니다.

**수학적 정의**: 확률 분포 $P$에 대한 $Q$의 Kullback-Leibler Divergence $D_{KL}(P||Q)$는 다음과 같습니다.

$$ D_{KL}(P||Q) = \sum_{x} P(x) \log_2 \frac{P(x)}{Q(x)} $$

*   $D_{KL}(P||Q) \ge 0$ 이며, $D_{KL}(P||Q) = 0$은 $P$와 $Q$가 완전히 동일한 분포일 때만 성립합니다.
*   KLD는 '거리'라는 용어를 사용하지만, 수학적인 거리 측정 기준(metric)은 아닙니다.

### 2.2. KLD의 비대칭성

KLD는 비대칭적입니다. 즉, $D_{KL}(P||Q) \ne D_{KL}(Q||P)$ 입니다.

*   $D_{KL}(P||Q)$는 $P$가 0이 아닌데 $Q$가 0인 경우에 무한대($\infty$)가 됩니다. 이는 $P$에서 발생 가능한 사건을 $Q$가 전혀 예측하지 못할 때 큰 페널티를 부여합니다. (모델 $Q$가 실제 $P$를 놓치지 않도록 강제)
*   $D_{KL}(Q||P)$는 $Q$가 0이 아닌데 $P$가 0인 경우에 무한대($\infty$)가 됩니다. 이는 $Q$가 실제 $P$에는 없는 사건을 예측할 때 큰 페널티를 부여합니다. (모델 $Q$가 불필요한 예측을 하지 않도록 강제)

이러한 비대칭성은 생성 모델(Generative Models)에서 $P$와 $Q$의 역할을 어떻게 정의하느냐에 따라 다른 최적화 결과를 가져올 수 있습니다.

### 2.3. 교차 엔트로피와의 관계 및 최소화 의미

KLD와 교차 엔트로피는 다음과 같은 관계를 가집니다.

$$ D_{KL}(P||Q) = H(P, Q) - H(P) $$

여기서 $H(P)$는 실제 분포 $P$의 엔트로피입니다. 모델을 학습하는 과정에서 실제 데이터 분포 $P$는 고정되어 있으므로 $H(P)$는 상수입니다.

따라서, $D_{KL}(P||Q)$를 최소화하는 것은 $H(P, Q)$를 최소화하는 것과 동일합니다.

$$ \text{Minimize } D_{KL}(P||Q) \iff \text{Minimize } H(P, Q) $$

이는 머신러닝에서 모델의 예측 분포 $Q$가 실제 데이터 분포 $P$에 최대한 가까워지도록 학습하는 것이 KLD를 최소화하는 것이며, 이는 곧 교차 엔트로피를 최소화하는 것과 같다는 의미입니다.

### 2.4. 스팸/정상 이메일 분류 사례의 분포 차이(바이너리/가우시안)를 통한 KLD 활용 방법

KLD는 분류 문제뿐만 아니라, 두 확률 분포 간의 차이를 측정해야 하는 다양한 머신러닝 시나리오에서 활용될 수 있습니다.

**사례: 스팸/정상 이메일 분류**

이메일이 스팸인지 정상인지 분류하는 모델을 개발한다고 가정해 봅시다. 각 이메일은 특정 단어의 출현 빈도(예: "무료", "당첨", "광고")와 같은 특성으로 표현될 수 있습니다.

1.  **바이너리 분류 (로지스틱 회귀)**:
    *   실제 이메일의 스팸 여부($P$)는 베르누이 분포(스팸/정상)를 따릅니다.
    *   모델이 예측하는 스팸 확률($Q$) 또한 베르누이 분포를 따릅니다.
    *   이 경우, 모델의 예측 분포 $Q$가 실제 분포 $P$를 얼마나 잘 근사하는지 KLD를 통해 측정할 수 있습니다. KLD를 최소화하는 것은 로지스틱 회귀의 교차 엔트로피 손실을 최소화하는 것과 동일합니다.

2.  **생성 모델 (Generative Models)에서의 KLD**:
    *   만약 이메일의 단어 출현 빈도가 가우시안 분포와 같은 연속 분포를 따른다고 가정한다면, KLD는 실제 단어 분포 $P$와 모델이 학습한 단어 분포 $Q$ 간의 차이를 측정하는 데 사용될 수 있습니다.
    *   예를 들어, 정상 이메일의 단어 분포 $P_{\text{정상}}$과 스팸 이메일의 단어 분포 $P_{\text{스팸}}$이 있다고 할 때, 모델은 이 두 분포를 학습하여 새로운 이메일이 어느 분포에 더 가까운지 판단할 수 있습니다.
    *   KLD는 변분 오토인코더(Variational Autoencoders, VAE)나 생성적 적대 신경망(Generative Adversarial Networks, GAN)과 같은 생성 모델에서 실제 데이터 분포와 모델이 생성한 데이터 분포 간의 차이를 최소화하는 데 핵심적인 역할을 합니다. 예를 들어, VAE의 손실 함수에는 잠재 공간(latent space)의 분포가 사전 분포(prior distribution)에 얼마나 가까운지를 측정하는 KLD 항이 포함됩니다.

## 3. 로지스틱 회귀에서의 정보이론적 해석

로지스틱 회귀는 분류 문제에 사용되는 선형 모델이며, 그 학습 과정은 정보이론적 관점에서 매우 명확하게 해석될 수 있습니다.

1.  **우도 최대화 (Maximum Likelihood Estimation, MLE)**:
    *   로지스틱 회귀의 파라미터($\mathbf{w}, b$)는 주어진 훈련 데이터가 관측될 확률(우도)을 최대화하는 방식으로 학습됩니다.
    *   이는 모델이 훈련 데이터의 실제 라벨을 가장 잘 설명하는 확률 분포를 학습하도록 하는 것입니다.

2.  **교차 엔트로피 최소화 (Cross-Entropy Minimization)**:
    *   우도 함수를 최대화하는 것은 음의 로그 우도 함수를 최소화하는 것과 동일합니다.
    *   로지스틱 회귀에서 이 음의 로그 우도 함수는 정확히 교차 엔트로피 손실 함수와 같습니다.
    *   따라서 로지스틱 회귀는 교차 엔트로피 손실을 최소화하는 방향으로 파라미터를 업데이트합니다.

3.  **Kullback-Leibler Divergence 최소화 (KLD Minimization)**:
    *   앞서 설명했듯이, 교차 엔트로피를 최소화하는 것은 KLD를 최소화하는 것과 동일합니다 (실제 분포의 엔트로피는 상수이므로).
    *   즉, 로지스틱 회귀는 훈련 데이터를 통해 학습된 모델의 예측 확률 분포 $Q$가 실제 데이터의 확률 분포 $P$에 최대한 가까워지도록 KLD를 최소화하는 것입니다.

**결론**: 로지스틱 회귀의 학습 목표는 정보이론적 관점에서 볼 때, 모델이 예측하는 확률 분포가 실제 데이터의 확률 분포와 가장 유사해지도록 만드는 것입니다. 이는 최대 우도 추정, 교차 엔트로피 최소화, 그리고 Kullback-Leibler Divergence 최소화라는 세 가지 관점에서 모두 동일한 목표를 지향하며, 이들이 서로 깊이 연결되어 있음을 보여줍니다. 이러한 정보이론적 이해는 머신러닝 모델의 작동 원리를 심층적으로 파악하고, 더 나아가 새로운 모델과 손실 함수를 설계하는 데 중요한 통찰력을 제공합니다.
