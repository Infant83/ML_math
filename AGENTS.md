# AGENTS.md

이 저장소는 머신러닝을 위한 기초 수학 강의자료를 장기적으로 다듬는 작업 공간입니다. 이후 에이전트나 협업자가 작업할 때는 아래 방침을 우선합니다.

## 기본 원칙

- 기본 응답과 문서 작성 언어는 한국어로 합니다.
- 작업 전 `git status -sb`로 현재 변경사항을 확인하고, 사용자가 만든 변경을 임의로 되돌리지 않습니다.
- Lecture-1/2 폴더의 Markdown 강의노트를 우선 원천 텍스트로 봅니다. HTML, PDF, DOCX는 배포/검토용 산출물로 취급합니다.
- 한국어 폴더명과 기존 파일명 체계를 유지합니다. 대규모 파일 이동이나 이름 변경은 사용자가 명시적으로 요청한 경우에만 수행합니다.
- 루트의 PPTX, DOCX, 노트북, 이미지 파일은 강의 초안 자산입니다. 삭제하거나 덮어쓰기 전에 반드시 의도를 확인합니다.

## 강의노트 작성 기준

- 강의식 흐름을 유지합니다: 학습 목표, 직관, 수학적 정의, 머신러닝 연결, 예제, 체크포인트 순서가 기본 골격입니다.
- 공식만 나열하지 말고, 해당 개념이 모델 학습, 손실함수, 결정경계, 확률추정, 평가 지표와 어떻게 이어지는지 설명합니다.
- 코드 예제는 재현 가능성을 우선합니다. 필요한 라이브러리, 데이터 생성 또는 데이터 출처, 실행 순서를 명확히 둡니다.
- 공개용 본문에는 내부 작업 흔적이나 초안 생성 과정 설명을 그대로 남기지 않습니다. 필요한 경우 "강사용 메모"나 자연스러운 본문 설명으로 바꿉니다.
- 수식 표기는 Markdown/LaTeX 렌더링을 고려해 작성하고, HTML/PDF 변환 시 깨지기 쉬운 표와 수식은 별도로 확인합니다.

## 파일 업데이트 기준

- 강의 내용 수정은 가능한 한 Markdown 원천 파일에서 시작합니다.
- 렌더링 산출물을 갱신할 때는 같은 장의 HTML/PDF/DOCX가 서로 같은 내용을 담도록 맞춥니다.
- 새 장을 추가할 때는 `chapterN_topic_lecturenote.md` 또는 기존 Lecture-2의 `_lecture_notes.md` 패턴 중 해당 폴더의 관례에 맞춥니다.
- README.md는 저장소의 강의 로드맵입니다. 장 추가, 주요 파일명 변경, 강의 범위 변경이 있으면 README도 함께 갱신합니다.
- AGENTS.md는 작업 방침입니다. 반복되는 운영 규칙이나 사용자의 명시적 선호가 새로 생기면 이 파일에 반영합니다.

## GitHub Pages 디자인 기준

- 루트 `index.html`은 강의 공유용 정적 Pages 진입점입니다.
- 페이지 디자인은 `https://infant83.github.io/GitLab-Onboarding-Lectures/index.html#route`의 톤을 기준으로 맞춥니다.
- 베이지 계열 배경, 얕은 그리드 질감, `Newsreader` 계열 세리프 헤딩, `page-shell` 중심 레이아웃, 원형 brand mark, pill 버튼, compact route/chapter card 구조를 유지합니다.
- 홈에서는 학습 관점 선택 UI보다 강의 챕터를 바로 보여 주는 구조를 우선합니다.
- 각 챕터는 `기하`, `최적화`, `확률`, `정보이론` 등 실제 주제 축에 매핑하고, 이 매핑을 홈 카드와 상세 페이지에 함께 반영합니다.
- `chapters/<chapter-id>/index.html` 상세 페이지는 기존 HTML 산출물을 그대로 붙여 넣는 공간이 아니라, 사이트 디자인에 맞춘 학술적 serving layer입니다.
- 상세 페이지에는 최소한 학습목표, 핵심 질문, 핵심 수식과 해석, 강의 진행 방식, 보강 audit, 원천 자료 링크가 있어야 합니다.
- 강의 페이지는 마케팅 랜딩 페이지가 아니라 학습 목표와 진행 순서가 바로 보이는 강의용 인터페이스로 유지합니다.

## 검토와 품질 확인

- Markdown 변경 후 제목 구조가 자연스러운지 `rg -n "^#{1,3} "`로 빠르게 확인합니다.
- 링크가 있는 문서를 수정했다면 상대경로가 실제 파일을 가리키는지 확인합니다.
- HTML/PDF/DOCX를 새로 생성한 경우, 최소 한 번은 열람 또는 파일 크기/생성일을 확인해 빈 산출물이 아닌지 봅니다.
- 큰 이론 설명을 고칠 때는 예제와 결론이 같은 메시지를 말하는지 점검합니다.

## Git 작업

- 커밋 전 `git diff -- README.md AGENTS.md`처럼 변경 범위를 확인합니다.
- 사용자 요청 없이 무관한 파일을 정리하거나 포맷하지 않습니다.
- 커밋 메시지는 변경 의도를 짧고 분명하게 씁니다.
- push 요청이 있으면 현재 브랜치와 원격을 확인한 뒤 push합니다. PR 생성은 사용자가 별도로 요청한 경우에만 진행합니다.
